# Document Name: [Coding/Determinants] Fr

## Aliases / Search Terms

- Fr
- form-reflection
- reflection secondary

## Core Definition

`Fr` is coded when form organizes the response first and the person also describes the object as a reflection or mirror image.  
Symmetry has to be involved, and the idea of reflection must be explicitly present.

## Application Conditions

- Form has to identify the main object first.
- The person must talk about reflection, mirroring, or an equivalent idea.
- The two objects must be explicitly or implicitly identical.
- If there are only two similar objects because of symmetry, without reflection language, reconsider [`(2)`](ref://scoring-input/pair).

## Cautions / Distinctions

- `Fr` is not the same as [`rF`](ref://scoring-input/determinants/rF); in `Fr`, form stays more clearly dominant.
- `Fr` does not replace the pair code; when there is reflection, do not also add [`(2)`](ref://scoring-input/pair).
- If the response only says there are two matching figures, but does not describe reflection or mirroring, `Fr` does not fit.
- If the response depends more on the reflective effect than on the form of the object, review [`rF`](ref://scoring-input/determinants/rF).
