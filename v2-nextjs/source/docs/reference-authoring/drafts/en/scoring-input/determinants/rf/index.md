# Document Name: [Coding/Determinants] rF

## Aliases / Search Terms

- rF
- reflection-form
- reflection dominant

## Core Definition

`rF` is coded when the impression of reflection or mirror image dominates the response and form stays secondary.  
Symmetry is still required, but the main emphasis falls on reflectiveness itself.

## Application Conditions

- The person must explicitly describe a reflection, mirror, or reflected image.
- The reflective impression must organize the response first.
- Form is still present, but with secondary weight.
- If there are only two matching objects because of symmetry, without reflection language, reconsider [`(2)`](ref://scoring-input/pair).

## Cautions / Distinctions

- `rF` is not the same as [`Fr`](ref://scoring-input/determinants/Fr); in `rF`, the reflection idea carries more weight.
- `rF` is also not the same as [`(2)`](ref://scoring-input/pair); pair requires symmetric identity, but not necessarily a reflected image.
- If the concrete form of the object is clearer than the reflective quality, `Fr` may fit better.
- Reflection is coded within determinants and is not combined with the pair code.
