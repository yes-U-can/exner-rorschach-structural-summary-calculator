# Document Name: [Coding/Determinants] FV

## Aliases / Search Terms

- FV
- form-vista
- vista secondary

## Core Definition

`FV` is coded when form organizes the response and depth or dimensionality stays secondary.  
The sense of distance or relief is present, but the percept still rests first on form.

## Application Conditions

- Form identifies the main object.
- The person then adds depth, distance, or relief.
- The dimensional quality must be genuinely stated in the response.
- If vista carries more weight than form, reconsider [`VF`](ref://scoring-input/determinants/VF).

## Cautions / Distinctions

- `FV` is not the same as [`VF`](ref://scoring-input/determinants/VF); the issue is which aspect dominates.
- If texture is the main experience, review [`FT`](ref://scoring-input/determinants/FT).
- If vague shaded discomfort is the main experience, review [`Y`](ref://scoring-input/determinants/Y).
- The interpretive use of this family returns later in [`SumV`](ref://result-interpretation/lower-section/core/SumV).
