# Document Name: [Coding/Determinants] VF

## Aliases / Search Terms

- VF
- vista-form
- vista dominant

## Core Definition

`VF` is coded when depth or dimensionality dominates the response and form stays secondary.  
There is still some form support, but the dimensional quality carries more weight than the formal structure.

## Application Conditions

- The main impression is depth, distance, or relief.
- Form participates only secondarily.
- If form ends up organizing the response, reconsider [`FV`](ref://scoring-input/determinants/FV).
- If formal support is barely present, reconsider [`V`](ref://scoring-input/determinants/V).

## Cautions / Distinctions

- `VF` should not be confused with [`TF`](ref://scoring-input/determinants/TF); one points to dimension and the other to texture.
- If tactile surface quality is more central, review `TF`.
- If the response depends more on diffuse shaded vagueness, review [`Y`](ref://scoring-input/determinants/Y).
- The interpretive use of this family returns later in [`SumV`](ref://result-interpretation/lower-section/core/SumV).
