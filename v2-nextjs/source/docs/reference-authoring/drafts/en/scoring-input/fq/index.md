# Document Name: [Coding/Form Quality] FQ

## Aliases / Search Terms

- Form Quality
- FQ
- `+`, `o`, `u`, `-`, `none`

## Core Definition

Form Quality (`FQ`) codes how well the described object fits the contours of the blot.
It is about formal fit, not about the organizational level of the response.

## Application Conditions

- Place the FQ symbol at the end of the determinant block.
- Use the five-code set: `+`, `o`, `u`, `-`, `none`.
- Base the decision on the form quality table first, then on extrapolation and fit judgment when needed.
- Ask whether the object can be seen quickly and plausibly without forcing the blot contours.
- Use [`none`](ref://scoring-input/fq/none) only when no codable form quality should be assigned.

## Cautions / Distinctions

- `FQ` is not the same as [`DQ`](ref://scoring-input/dq). `DQ` is about organization; `FQ` is about form fit.
- `FQ+` is uncommon. A response is not `+` just because it sounds rich or clever.
- The most delicate distinction outside the table is often between [`u`](ref://scoring-input/fq/u) and [`-`](ref://scoring-input/fq/-).
- `none` does not mean poor form. It means no form quality code should be assigned.
