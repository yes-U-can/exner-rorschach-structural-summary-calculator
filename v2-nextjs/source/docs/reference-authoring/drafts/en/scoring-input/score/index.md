# Document Name: [Coding] score

## Aliases / Search Terms

- score
- coding
- response scoring
- code a response

## Core Definition

`score` does not mean assigning an immediate numeric point value.
In the Comprehensive System, it means converting each Rorschach response into a precise combination of codes.

## Application Conditions

- Code each response in the order it appears in the protocol.
- Coding should fix location, developmental quality, determinants, form quality, contents, and additional codes when needed.
- Inquiry should clarify the response enough so that coding represents what the person actually saw.
- Structural Summary accuracy depends on consistent response coding.

## Cautions / Distinctions

- `score` is not a separate numeric scale outside the protocol; it is the full response-coding process.
- Do not guess a code just because it seems likely. If inquiry does not support it, do not assign it.
- Coding and interpretation are separate stages. First code the response; then interpret the protocol results.
- Coding components are decided through different routes, such as [`Location`](ref://scoring-input/location), [`DQ`](ref://scoring-input/dq), [`Determinants`](ref://scoring-input/determinants), [`FQ`](ref://scoring-input/fq), and [`Contents`](ref://scoring-input/contents).
