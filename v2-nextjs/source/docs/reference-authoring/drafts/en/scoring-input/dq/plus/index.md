# Document Name: [Coding/Developmental Quality] +

## Aliases / Search Terms

- DQ+
- synthesized response (+)
- synthesis with formal demand

## Core Definition

`DQ+` is assigned when the response synthesizes two or more separate but meaningfully related objects, and at least one of those objects requires specific form demand.
The key is not just “more than one object,” but an organized whole.

## Application Conditions

- More than one distinct object or part is present in the response.
- The objects are meaningfully related to each other.
- At least one involved object requires specific form demand.
- The synthesis feels organized rather than merely listed side by side.

## Cautions / Distinctions

- Two objects that only mirror each other, without meaningful integration, usually do not justify `DQ+`; that pattern more often fits [`DQo`](ref://scoring-input/dq/o).
- If synthesis is present but none of the objects requires specific form demand, use [`DQv/+`](ref://scoring-input/dq/v/%2B).
- `DQ+` is not the same as [`FQ+`](ref://scoring-input/fq/%2B). `DQ+` is about organization; `FQ+` is about form quality.
