# Nombre del documento: [Codificacion/Calidad evolutiva] +

## Alias y busqueda

- DQ+
- respuesta de sintesis (+)
- sintesis con demanda formal

## Definicion central

DQ+ se asigna cuando la respuesta sintetiza dos o mas objetos separados pero relacionados y, ademas, por lo menos uno de ellos exige una forma especifica.  
La clave no es solo que haya varios objetos, sino que la respuesta forme una unidad organizada.

## Condiciones de aplicacion

- Hay mas de un objeto o parte diferenciada dentro de la respuesta.
- Los objetos aparecen en una relacion significativa entre si.
- Por lo menos uno de los objetos implica demanda formal especifica.
- La sintesis debe sentirse estructurada, no simplemente yuxtapuesta.

## Precauciones y distinciones

- Dos objetos parecidos por simetria, sin una relacion significativa, no bastan para `DQ+`; en ese caso suele corresponder [`DQo`](ref://scoring-input/dq/o).
- Si existe sintesis, pero ninguno de los objetos exige forma especifica, corresponde [`DQv/+`](ref://scoring-input/dq/v/%2B).
- `DQ+` no significa lo mismo que [`FQ+`](ref://scoring-input/fq/%2B). El primero habla de organizacion; el segundo de calidad formal.
