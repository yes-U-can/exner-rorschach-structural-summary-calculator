# Nombre del documento: [Codificacion/Calidad evolutiva] o

## Alias y busqueda

- DQo
- respuesta ordinaria
- ordinaria con demanda formal

## Definicion central

DQo se usa cuando la respuesta no alcanza una sintesis significativa, pero el objeto identificado si exige una forma especifica.  
Puede tratarse de un solo objeto claramente definido o de una respuesta con varios objetos sin una relacion significativa suficiente para recibir un codigo de sintesis.

## Condiciones de aplicacion

- El objeto descrito exige una forma especifica, ya sea por su clase o por la manera en que el sujeto lo describe.
- No hay una sintesis significativa entre objetos separados.
- La respuesta puede ser simple o puede incluir varios objetos sin integracion suficiente.
- Si la descripcion verbal obliga a pensar en una forma concreta, `o` sigue siendo la opcion principal.

## Precauciones y distinciones

- Si existe una relacion significativa entre objetos separados y al menos uno exige forma especifica, corresponde [`DQ+`](ref://scoring-input/dq/%2B).
- Si el objeto no exige forma especifica y tampoco hay sintesis, corresponde [`DQv`](ref://scoring-input/dq/v).
- No confunda `DQo` con [`FQo`](ref://scoring-input/fq/o). DQ y FQ codifican aspectos distintos.
