# Nombre del documento: [Codificación/Calidad evolutiva] o

## Alias y búsqueda

- DQo
- respuesta ordinaria
- ordinaria con demanda formal

## Definición central

DQo se usa cuando la respuesta no alcanza una síntesis significativa, pero el objeto identificado si exige una forma específica.
Puede tratarse de un solo objeto claramente definido o de una respuesta con varios objetos sin una relación significativa suficiente para recibir un código de síntesis.

## Condiciones de aplicación

- El objeto descrito exige una forma específica, ya sea por su clase o por la manera en que el sujeto lo describe.
- No hay una síntesis significativa entre objetos separados.
- La respuesta puede ser simple o puede incluir varios objetos sin integración suficiente.
- Si la descripción verbal obliga a pensar en una forma concreta, `o` sigue siendo la opción principal.

## Precauciones y distinciones

- Si existe una relación significativa entre objetos separados y al menos uno exige forma específica, corresponde [`DQ+`](ref://scoring-input/dq/%2B).
- Si el objeto no exige forma específica y tampoco hay síntesis, corresponde [`DQv`](ref://scoring-input/dq/v).
- No confunda `DQo` con [`FQo`](ref://scoring-input/fq/o). DQ y FQ codifican aspectos distintos.
