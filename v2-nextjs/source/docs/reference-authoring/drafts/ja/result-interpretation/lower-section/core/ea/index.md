# [Interpretation/Core] EA

## 別名・検索語

- EA
- Experience Actual
- available resources
- 利用可能資源

## 中核的定義

`EA` は `EB` を構成する成分から導かれる、利用可能な資源量の指標です。
日常的な要求に対処し、現在の体験を調整するための有効な容量がどの程度あるかを示します。

## 解釈の要点

- 十分な `EA` は、日常的な要求に対応するための基本資源があることを示唆します。
- 低い `EA` は、緊張調整や要求処理の資源が限られている可能性を示します。
- `EA` が十分でも、それだけで適応が保証されるわけではなく、[`es`](ref://result-interpretation/lower-section/core/es) と比較して読む必要があります。
- `EA` は [`D`](ref://result-interpretation/lower-section/core/D) と [`AdjD`](ref://result-interpretation/lower-section/core/AdjD) を読むための主要な土台です。

## 注意点・鑑別

- `EA` が十分に見えても、[`EBPer`](ref://result-interpretation/lower-section/core/EBPer) が非常に硬い場合や [`es`](ref://result-interpretation/lower-section/core/es) が非常に高い場合には、実際の機能は不安定になりえます。
- 短いプロトコルや高い [`Lambda`](ref://result-interpretation/lower-section/core/Lambda) のプロトコルでは、この値が現実を単純化しすぎることがあります。
- `EA` は「潜在的に使える資源」を示すのであって、あらゆる状況で完全に使えていることを意味しません。
