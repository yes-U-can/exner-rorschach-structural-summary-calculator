# 문서명: [해석] result-interpretation

## 별칭/검색어

- result-interpretation
- interpretation
- 해석
- 구조요약 해석

## 핵심 정의

`result-interpretation`은 부호화가 끝난 뒤 만들어진 구조요약 자료를 바탕으로 심리적 기능을 해석하는 영역 전체를 가리킨다. 이 문서는 `개별 반응을 어떻게 코드화할까`가 아니라, `정리된 변수 묶음을 어떻게 읽을까`를 안내하는 상위 개요다.

## 해석 포인트

- 해석은 먼저 정확한 `구조요약`을 전제로 한다. 부호화가 흔들리면 해석도 흔들린다.
- 해석은 보통 `upper section`, `lower section`, `special indices`처럼 cluster 단위로 진행한다.
- 단일 변수 하나만 보고 결론을 확정하지 않는다. 대부분의 변수는 다른 변수들과 함께 볼 때 의미가 선명해진다.
- 구조요약은 대표적 자료이지만, 필요하면 반응 sequence와 내용 정교화도 함께 본다.
- 문화권, 연령, 성별, 상황 맥락에 따라 같은 수치의 의미가 달라질 수 있으므로, 구조요약 수치는 면담과 행동관찰을 함께 고려해 해석한다.

## 주의사항/감별 기준

- 해석 문서는 `부호화 문서`와 분리해서 읽어야 한다.
- `점수가 높다/낮다`는 사실만으로 진단을 바로 내리면 안 된다.
- 같은 변수라도 referral question과 전체 profile 맥락 안에서 의미가 달라질 수 있다.
- 구조요약은 중요하지만, 상징적 내용만 보거나 반대로 수치만 보는 식으로 한쪽에 치우치지 않는다.
