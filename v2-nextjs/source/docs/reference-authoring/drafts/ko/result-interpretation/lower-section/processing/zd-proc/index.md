# 문서명: [해석/Processing] Zd

## 별칭/검색어

- Zd
- Processing efficiency
- 처리능력

## 핵심 정의

`Zd`는 실제 조직화 양과 기대 조직화 양의 차이를 보여 주는 점수다. 처리 구역에서는 스캔과 조직화가 기대 수준보다 지나치게 많았는지, 적었는지, 혹은 대체로 적절했는지를 읽는다.

## 해석 포인트

- 종합체계 기준으로 `Zd`가 대체로 `+3.0`과 `-3.0` 사이에 있으면 처리 효율이 보통 범위라고 본다.
- 큰 양수는 과다 포섭, 과도한 점검, 지나친 신중함 가능성을 시사한다.
- 큰 음수는 과소 포섭, 서두름, 충분히 훑지 않고 넘어가는 경향 가능성을 시사한다.
- `PSV`가 없고 `Zd`가 보통 범위이면 처리 효율은 대체로 안정적이라고 볼 수 있다.

## 높거나 낮을 때 주의

- `Zd`가 양수라고 늘 좋은 것은 아니다. 지나친 점검은 결정 지연이나 경직성과 연결될 수 있다.
- `Zd`가 음수라고 늘 나쁜 것도 아니다. 다만 서두름이나 단순화 경향과 함께 볼 필요가 있다.
- `PSV`가 있으면 `Zd`가 평균 범위여도 효율 문제를 다시 본다.

## 단독 해석 한계

- `Zd`는 효율의 한 축일 뿐, 처리의 폭이나 질 전체를 대신하지 않는다.
- 반드시 `PSV`, `Zf`, `W:D:Dd`와 함께 읽어야 한다.

## 함께 볼 변수

- `PSV`
- `Zf`
- `W:D:Dd`
- `DQ+`
- `DQv`
