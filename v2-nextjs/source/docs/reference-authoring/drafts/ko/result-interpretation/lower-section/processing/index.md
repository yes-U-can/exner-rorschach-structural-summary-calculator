# 문서명: [해석/Lower Section/Processing] Processing

## 별칭/검색어

- Processing
- Information Processing
- 처리
- 정보처리

## 핵심 정의

`Processing`은 피검자가 반점 자극을 얼마나 살피고, 어떤 범위로 훑고, 어느 정도 복잡하게 조직하여 입력하는지를 읽는 묶음이다. 인지 삼제에서 `Processing`은 입력 단계에 해당하며, 그 다음 단계인 `Mediation`과 `Ideation`의 토대를 이룬다.

## 해석 포인트

- 이 구역은 자극을 많이 보는지, 덜 보는지, 넓게 보는지, 세부로 가는지, 복잡하게 조직하는지, 그리고 그 과정이 효율적인지를 함께 읽는다.
- `Zf`, `Zd`, `W:D:Dd`, `W:M`, `PSV`, `DQ+`, `DQv`는 서로 다른 각도에서 처리 전략을 보여 준다.
- 같은 low-quality response라도, 입력 단계에서 무너졌는지(`Processing`), 번역 단계에서 틀어졌는지(`Mediation`), 개념화 단계에서 흔들렸는지(`Ideation`)를 구분하는 것이 중요하다.

## 높거나 낮을 때 주의

- 값이 높다고 항상 좋고, 낮다고 항상 나쁘다고 읽지 않는다.
- 더 복잡한 처리와 더 효율적인 처리는 같은 뜻이 아니다.
- 기록 길이, 방어성, 연령, 최근 스트레스는 처리 변수에 직접 영향을 준다.

## 단독 해석 한계

- `Processing`만으로 전체 인지기능을 결론내리면 오독 위험이 크다.
- 반드시 `Mediation`, `Ideation`, `FQ`, `Special Scores`, `Lambda`와 함께 읽어야 한다.
