# 문서명: [해석/Lower Section/Core] Core

## 별칭/검색어

- core
- 핵심
- 통제와 스트레스 핵심

## 핵심 정의

`Core`는 기록의 기본 자원, 대처 스타일, 현재 요구량, 스트레스 견딤, 긴장 자극을 읽는 하부 핵심 묶음이다. 해석에서는 `R`, `Lambda`, `EB`, `EA`, `eb`, `es`, `D`, `AdjD` 등을 함께 보며, 기록이 실제로 얼마나 신뢰롭게 사람의 일상 기능을 드러내는지부터 점검한다.

## 해석 포인트

- `R`은 기록 길이와 기본 해석 가능성을 본다.
- `Lambda`는 주의 폭과 방어적 단순화 여부를 본다.
- `EB`, `EA`, `eb`, `es`, `D`, `AdjD`는 자원과 요구의 균형을 본다.
- `FM`, `m`, `SumCprime`, `SumT`, `SumV`, `SumY`는 긴장과 부담의 성격을 세분해 준다.

## 높거나 낮을 때 주의

- `Core` 변수는 서로 영향을 준다. 하나만 높거나 낮다고 바로 강점/약점으로 결정하지 않는다.
- 짧은 기록이나 높은 `Lambda`는 `EA`, `D`, `AdjD` 같은 값들을 과소추정하거나 왜곡할 수 있다.

## 단독 해석 한계

- `Core`는 구조 전체의 발판이다.
- 이 묶음이 불안정하면 이후 cluster 해석도 보수적으로 읽어야 한다.

## 함께 볼 변수

- `R`
- `Lambda`
- `EB`
- `EA`
- `eb`
- `es`
- `D`
- `AdjD`
