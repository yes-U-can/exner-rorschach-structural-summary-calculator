# 문서명: [해석/Upper Section] Zd

## 별칭/검색어

- Zd
- Z difference
- 조직화 효율 차이

## 핵심 정의

`Zd`는 실제 조직화 양(`ZSum`)이 기대치(`ZEst`)보다 얼마나 많거나 적은지를 보여주는 차이 점수다. 상부 영역에서는 조직화 효율과 처리 속도의 균형을 읽을 때 가장 직접적으로 쓰이는 값이다.

## 해석 포인트

- `Zd`가 0 부근이면 대체로 기대 수준과 비슷한 조직화 양을 뜻한다.
- 양의 방향으로 크게 가면 과다 조직화, 과도한 점검, 지나치게 많은 정보 결합 가능성을 생각한다.
- 음의 방향으로 크게 가면 과소 조직화, 서두름, 덜 살피고 결론 내리는 경향 가능성을 생각한다.

## 높거나 낮을 때 주의

- 큰 양수는 흔히 `overincorporation` 쪽 경고로 읽지만, 단순히 꼼꼼함과 어떻게 다른지 다른 변수로 다시 확인해야 한다.
- 큰 음수는 흔히 `underincorporation` 쪽 경고로 읽지만, 피로, 저동기, 짧은 기록, 충동성 중 무엇인지 분리해야 한다.
- 절단값 근처 해석은 기록 전체 맥락을 더 중시한다.

## 단독 해석 한계

- `Zd`는 중요하지만 하나만으로 병리 결론을 내리면 안 된다.
- `Lambda`, `W:D:Dd`, `PSV`, `DQ 분포`, `XA%`류와 함께 볼 때 의미가 더 분명해진다.

## 함께 볼 변수

- `ZSum`
- `ZEst`
- `PSV`
- `Zd_proc`
