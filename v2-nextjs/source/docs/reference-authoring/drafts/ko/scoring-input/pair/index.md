# 문서명: [부호화/쌍반응] Pair

## 별칭/검색어

- Pair
- (2)
- pair response
- 쌍반응

## 핵심 정의

`(2)`는 반점의 대칭성 때문에 동일한 두 대상이 지각되었다고 보고할 때 붙이는 쌍반응 coding이다.

## 채점/적용 조건

- 피검자가 대칭성을 바탕으로 `둘`, `한 쌍`, `하나는 여기 하나는 저기`처럼 두 개의 동일한 대상을 말하면 `(2)`를 검토한다.
- 핵심은 대칭성 때문에 두 대상을 본다는 점이다.
- 형태특정성이나 형태질과는 무관하게, 대칭적 쌍지각이 확인되면 `(2)`를 붙일 수 있다.
- 사용된 위치영역이 실제로 대칭성과 관련되는지 확인하는 것이 중요하다.

## 주의사항/감별 기준

- 반사 개념이 직접 들어가면 `Fr` 또는 `rF`를 먼저 본다. 반사는 이미 두 대상이 보였음을 포함하므로 보통 `(2)`를 따로 쓰지 않는다.
- 단순히 두 개의 대상이 있다고 해서 모두 `(2)`는 아니다. 대칭성이 근거여야 한다.
- 서로 다른 위치영역에서 각각 하나씩 본 두 대상은 `(2)`가 아닐 수 있으므로 inquiry로 확인해야 한다.
- `(2)`는 나중에 자기지각 관련 비율과 연결되지만, coding 단계에서는 먼저 대칭적 쌍지각 여부만 본다.
