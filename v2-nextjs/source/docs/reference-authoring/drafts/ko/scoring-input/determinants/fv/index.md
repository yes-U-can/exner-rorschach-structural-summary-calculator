# 문서명: [부호화/결정인] FV

## 별칭/검색어

- FV
- form-vista
- 형태-vista

## 핵심 정의

FV는 형태가 먼저 대상을 조직하고, 음영 특징이 그 대상에 깊이감이나 입체감을 2차적으로 부여할 때 주는 vista determinant이다. 기본 형태는 비교적 구체적이지만, `뒤에 있다`, `밑에서 나온다`, `안쪽으로 들어가 보인다` 같은 차원성 정보가 추가된다.

## 채점/적용 조건

- 반응의 기본 대상이 형태로 먼저 분명하게 잡혀 있어야 한다.
- 음영 특징이 깊이, 거리, 가림, 입체감을 보강해야 한다.
- 질문 단계에서 피검자가 실제로 음영 차이 때문에 입체감이나 원근을 보았다고 확인해야 한다.
- vista 단서가 빠져도 기본 형태가 어느 정도 유지되는지 확인한다.

## 주의사항/감별 기준

- `FV`는 `VF`보다 형태 우세가 더 분명하다.
- `FV`는 `F`와 달리 차원성 정교화가 실제로 확인되어야 한다.
- `거칠다`, `부드럽다`, `털 같다`가 핵심이면 `FT/TF/T`를 먼저 검토한다.
- 막연한 어둡고 흐린 불편감이 핵심이면 `Y`를 먼저 검토한다.
- `뒤에 있다`, `가려져 있다`라는 말만으로 자동 coding 하지 말고, 그 근거가 음영에서 왔는지 확인한다.
