# 문서명: [부호화/결정인] FD

## 별칭/검색어

- FD
- form-dimension
- 형태-차원성

## 핵심 정의

FD는 음영이 아니라 형태만으로 3차원성, 조망, 원근이 성립할 때 주는 form-dimension determinant이다. `뒤로 기대고 있다`, `멀리 있어서 작아 보인다`처럼 차원성 해석이 있지만, 그 근거가 형태 관계 자체에 있어야 한다.

## 채점/적용 조건

- 반응이 형태만으로 차원성이나 조망을 성립시켜야 한다.
- 음영 차이 없이도 깊이, 거리, 위치관계를 설명할 수 있어야 한다.
- 질문 단계에서 피검자가 크기 차이, 위치 관계, 형태 배열을 근거로 차원성을 보았다고 확인해야 한다.
- 음영 차이가 핵심이면 `FD`가 아니라 `FV`, `VF`, `V`를 재검토한다.

## 주의사항/감별 기준

- `FD`는 `F`와 달리 형태만으로 3차원성이나 조망이 추가되어야 한다.
- `FD`는 `FV/VF/V`와 달리 음영이 아니라 형태 관계가 차원성의 근거다.
- `FD`는 내용 code `Fd`와 다르다. 하나는 determinant이고, 다른 하나는 content다.
- `뒤에 있다`, `누워 있다`, `멀리 있다` 같은 표현이 나와도 그 근거가 음영이면 `FD`가 아니다.
- 혼합 반응에서 `FD`가 다른 determinant와 함께 들어갈 수 있으므로, 형태차원 coding을 놓치지 않도록 Inquiry를 본다.
