# [부호화] scoring-input

## 별칭/검색어

- scoring-input
- scoring input
- coding input
- 부호화
- 채점 입력

## 핵심 정의

`scoring-input`은 피검자의 각 반응을 종합체계의 표준 기호로 바꾸어 구조요약에 넣기 전까지의 부호화 영역 전체를 가리킨다. 이 문서는 `어떤 값을 계산하느냐`가 아니라 `각 반응을 어떤 코드로 기록하느냐`를 안내하는 상위 개요다.

## 채점/적용 조건

- 부호화는 반응을 카드 순서와 반응 순서대로 하나씩 검토하면서 진행한다.
- 각 반응마다 먼저 `위치`, `발달질`, `결정인`, `형태질`, `내용`, `추가 coding 요소`를 확인한다.
- 필요하면 `Pair`, `Popular`, `Z`, `GHR/PHR`, `Special Scores` 같은 보조 coding을 추가한다.
- inquiry에서 확인된 정보만 coding에 사용한다.
- 부호화 단계의 목적은 뒤에서 `구조요약`을 정확히 계산할 수 있도록 반응을 일관된 symbol 세트로 기록하는 것이다.

## 주의사항/감별 기준

- 로샤의 부호화는 숫자 점수만 매기는 작업이 아니다.
- 이 단계에서는 `해석`보다 `정확한 기록`이 우선이다.
- 카드 특성이나 흔한 반응은 참고가 될 수 있지만, 어떤 code를 줄지는 언제나 현재 반응의 실제 표현을 기준으로 정한다.
- 같은 기호라도 문맥이 다르면 다른 문서로 봐야 한다.
  - 예: `카드 V`와 `결정인 V`
  - 예: `DQ +`와 `FQ +`
