# 문서명: [부호화/카드] Card

## 별칭/검색어

- Card
- cards
- 카드
- 로샤 카드

## 핵심 정의

`Card` 문서는 10장 각각의 카드가 부호화에서 어떤 주의점을 만드는지 정리하는 상위 개요다. 이 문서는 카드의 상징 해석이 아니라, `카드별 coding note`를 어디에 연결할지 보여주는 길잡이 페이지다.

## 채점/적용 조건

- 카드 문서는 특정 카드에서 `자주 나오는 반응`, `자주 헷갈리는 determinant/content`, `Popular 확인 포인트`를 짧게 안내한다.
- 실제 채점은 언제나 반응 그 자체를 기준으로 한다. 카드 특성은 coding을 돕는 참고틀일 뿐이다.
- 같은 내용이라도 카드에 따라 `Popular` 여부나 흔한 조직화 방식이 다를 수 있다.
- locale마다 카드별 설명의 본문과 relatedRoutes는 달라질 수 있지만, route skeleton은 공통이다.

## 주의사항/감별 기준

- 카드 문서를 읽었다고 해서 해당 카드 반응을 자동으로 특정 coding에 넣으면 안 된다.
- 카드 문서는 `상징 사전`이 아니라 `coding 보조 note`다.
- 카드 번호와 같은 문자 기호가 다른 coding 요소와 충돌할 수 있으므로 문맥을 항상 확인한다.
  - 예: `카드 V`와 `결정인 V`
