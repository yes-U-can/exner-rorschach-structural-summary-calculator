# v2.2.0 Workspace Shell and Chat UX Validation

Date: 2026-07-14 (KST)

Status: local pre-release verification pass

## Goal

Move the calculator into a consistent application shell and bring the interpretation assistant closer to established AI SaaS interaction patterns without changing scoring logic, the AI harness, RAG, the reference corpus, model selection, or the BYOK storage boundary.

## Scope

- Fixed 72 px desktop rail and collapsible 300 px overlay sidebar
- Mobile top bar and 310 px left drawer
- Calculator, interpretation assistant, reference, service, legal, version, and shortcut navigation
- One state-dependent AI session control plus an explicit session-end confirmation dialog
- Vertically stacked language, theme, and AI session controls
- Existing calculator shortcuts plus `Ctrl/Command + B` for the sidebar
- Cardless, full-height interpretation chat with an internal message scroller
- Glass-style prompt composer
- Untitled right-edge prompt navigation rail built only from user prompts
- Cardless reference search, reference reading, service, and legal document surfaces
- One shared page-background token across calculator, chat, reference, service, legal, and version routes
- Korean, English, Japanese, Spanish, and Portuguese UI parity
- Calculator row handles centered over the table's left boundary and layered above it, plus reliable plain, Shift, and Ctrl/Command row selection from the number cell
- Compact Google Apps Script archive guidance exposed through a keyboard-focusable information tooltip
- Fixed-model chat readiness that no longer depends on a second client-side model-catalog request

## Interaction Rules

- The calculator remains the primary workspace and the route back to scoring.
- The interpretation assistant retains the existing BYOK authorization gate.
- The desktop drawer overlays, dims, and blocks the workspace rather than pushing it.
- The persistent rail, drawer header, and navigation icons keep the same upper-left coordinates.
- Drawer labels fade before the closing edge reaches their text and appear after opening width is available; icons do not move during either transition.
- Desktop and mobile sidebars do not use an internal vertical scrollbar at the validated heights.
- Language, theme, and AI session controls use the same vertical order in all five locales.
- Logged-out state shows only the session-start action; logged-in state shows only the session-end action.
- Session end removes the temporary API-key session and ephemeral AI conversation only after confirmation.
- `Ctrl/Command + B` is the only new application-wide shortcut.
- The shortcut dialog documents existing `Ctrl/Command + Z`, redo, `Ctrl/Command + A`, `Shift + Click`, `Ctrl/Command + Click`, and `Ctrl/Command + J` behavior without inventing new commands.
- The shortcut dialog remains centered against the physical viewport while page scroll locking is released during close.
- The chat page has no outer card, no page-level conversation growth, and no nested page scroller.
- The chat scroller reaches the bottom of the workspace behind the floating composer; conversation bottom padding keeps the final message and actions visible above it.
- The jump-to-latest button and desktop prompt rail stay above the floating composer instead of overlapping it.
- The prompt navigation rail is shown at 1280 px and wider without reserving a separate content column; it is hidden below that breakpoint so it cannot squeeze mobile conversation content.
- Rail markers use user prompts only. The active marker is longer and uses the brand color; hovering or focusing previews the prompt text.
- Selecting a marker uses smooth `scrollIntoView` behavior, pauses automatic stream-following, and does not alter the conversation.
- The structural-summary control accepts multi-line clipboard content through an invisible textarea, keeps the raw string out of the visual UI, and shows only the localized empty or entered state.
- The original localized first-message examples remain semantically intact in all five locales, while the redundant `Example:`-style prefix is removed.
- The prompt composer uses existing color tokens, dark mode, an 8 px radius, and a browser-computed 32 px backdrop blur.
- Reference search uses the same form component on the index and detail routes. Search-result links preserve the trimmed query in the detail URL.
- Reference, service, terms, privacy, and version archive routes remain directly addressable pages; only decorative outer card framing was removed.
