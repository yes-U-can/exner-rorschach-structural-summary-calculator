## Documentation Standard

The standard makes these rules explicit:

- app-facing user experience can be localized into Korean, English, Japanese, Spanish, and Portuguese
- public release notes and CHANGELOG entries use Korean as the canonical record
- the public showcase entry point can provide an English companion overview for global GitHub visitors
- five-language release notes are not maintained by default because they create avoidable drift
- command names, file paths, API names, model IDs, test names, and generated evidence identifiers keep their original English form
- release notes use absolute `YYYY-MM-DD` dates
- release notes state whether UI/UX changed
- public evidence summarizes process and metadata, not raw prompts, raw model answers, API keys, private assessment payloads, or local runtime logs

## Privacy Boundary

This batch does not add raw AI prompts, raw model outputs, private assessment payloads, local runtime logs, `.env*` files, Vercel local state, or API keys.

The purpose is governance and release-readiness documentation, not new model evaluation output.
