## 먼저 내린 결론

기존 계산기가 통째로 잘못되어 있었던 것은 아니다. 반응 수, 위치, DQ, 형태질, 결정인, 내용, 특수점수, Z 합계, 주요 비율과 여러 특수지표는 독립 사례에서 대체로 일치했다.

그러나 “결함이 전혀 없었다”는 결론도 사실이 아니다. 이번 감사에서 일곱 가지 경계조건 또는 수식 결함을 확인했고 v2.2.0 코드에서 수정했다. 일부는 드문 입력에서만 나타나지만, 자기중심성 지표와 HVI처럼 해석에 영향을 줄 수 있는 항목도 있으므로 정식 버그 수정으로 취급한다.

이 감사는 기존 앱의 출력만 서로 비교하지 않았다. 다음 세 근거를 독립적으로 대조했다.

1. 초기 v1 GAS 개발에 참고한 공개 배포 2019 Excel workbook의 실제 셀 수식
2. 점수열과 구조요약 결과가 함께 공개된 외부 사례
3. 현행 TypeScript 구현과 경계값·순서불변성 자동 테스트

## 쉬운 수식 설명

### 1. WSumC와 EA

색채 반응은 통제 정도에 따라 서로 다른 가중치를 갖는다.

```text
WSumC = 0.5 × FC + 1.0 × CF + 1.5 × C
EA = M + WSumC
```

TypeScript 구현:

```ts
const WSumC = (0.5 * FC) + (1.0 * CF) + (1.5 * C);
const EA = M_total + WSumC;
```

### 2. es, AdjEs, D, AdjD

```text
es = FM + m + SumC' + SumT + SumV + SumY
AdjEs = es - max(0, m - 1) - max(0, SumY - 1)
D = D 표에서 EA - es가 속한 구간
AdjD = D 표에서 EA - AdjEs가 속한 구간
```

`D`는 단순 반올림이 아니라 정해진 구간표를 사용하는 값이다. 이번 테스트는 `-20`부터 `20`까지 `0.5` 간격의 모든 81개 값을 원래 구간표와 비교했다.

### 3. Lambda

```text
Lambda = 순수 F 반응 수 / (전체 반응 수 - 순수 F 반응 수)
```

모든 반응이 순수 F이면 분모가 0이므로 결과는 `0`이 아니라 무한대다. v2.2.0은 이 경우 `∞`를 표시한다.

```ts
const lambdaDenominator = R - F_pure;
const Lambda = lambdaDenominator === 0
  ? '∞'
  : F_pure / lambdaDenominator;
```

### 4. EBPer

EB의 큰 쪽을 작은 쪽으로 나눈 숫자를 언제나 보여주는 지표가 아니다.

```text
EA가 4 이상이고,
M과 WSumC가 모두 0보다 크며,
큰 값 / 작은 값이 2.5 이상일 때만 EBPer를 표시한다.
```

```ts
if (M_total > 0 && WSumC > 0) {
  const ratio = Math.max(M_total, WSumC) / Math.min(M_total, WSumC);
  if (EA >= 4 && ratio >= 2.5) EBPer = ratio;
}
```

### 5. 자기중심성 지표

2019 Excel 원본의 `Lower_Section!X22`와 표준식을 다시 대조했다.

```text
3r+(2)/R = [3 × (Fr + rF) + Pair] / R
```

여기서 `(2)`는 Pair 반응 수다. 기존 TypeScript에는 반사반응 계수가 `2`로 잘못 이식되어 있었고, v2.2.0에서 `3`으로 수정했다.

```ts
const EgocentricityIndex = R > 0
  ? (((Fr + rF) * 3 + F_pairs) / R)
  : 0;
```

### 6. 능동·수동 운동

`Ma-p`, `FMa-p`, `ma-p`는 능동과 수동 양쪽에 각각 한 번씩 들어간다. 문자열 일부 포함 여부로 세면 혼합코드가 누락될 수 있으므로, v2.2.0은 허용된 정확한 코드를 각각 합산한다.

```ts
const active = ['Ma', 'Ma-p', 'FMa', 'FMa-p', 'ma', 'ma-p'];
const passive = ['Mp', 'Ma-p', 'FMp', 'FMa-p', 'mp', 'ma-p'];
```

### 7. ZEst와 Zd

```text
ZSum = 카드와 Z 종류에 따른 가중치의 합
ZEst = Zf에 대응하는 표준 기대값
Zd = ZSum - ZEst
```

ZEst 표의 마지막 유효 입력인 `Zf=50`은 `173`이다. 기존 경계검사는 마지막 칸을 범위 밖으로 처리했으며 v2.2.0에서 수정했다.

## 독립 계산 정답지

### A. RIAP v5 공개 예제

- 반응 수: 20
- 검증 범위: 위치, DQ, FQ, 결정인, 내용, Z, 주요 비율, 여섯 특수지표
- 출처: [Essentials of Rorschach Assessment, sample computerized score reports](https://elmirmohammedmemorypsy.com/wp-content/uploads/2021/04/essentials-of-rorschach-assessment.pdf)

대표 일치값: `R=20`, `Lambda=.43`, `EA=9.5`, `Zf=12`, `ZSum=40.0`, `ZEst=38.0`, `Zd=2.0`, `DEPI=5 Positive`.

### B. Engelman et al. 공개 사례

- 반응 수: 20
- 검증 범위: 공개된 점수열과 Structural Summary
- 출처: [“Why am I so stuck?”: A Collaborative/Therapeutic Assessment Case Discussion](https://www.therapeuticassessment.com/docs/Engelman_et_al_2016_copy.pdf)

대표 일치값: `R=20`, `Lambda=.18`, `EA=7.5`, `Zf=18`, `ZSum=53.5`, `ZEst=59.5`, `Zd=-6.0`, `DEPI=6 Positive`.

논문의 표는 `HVI=Yes`로 인쇄되어 있지만, 같은 논문에 공개된 점수열은 HVI의 필수조건과 보조조건 네 개를 충족하지 않는다. 앱을 표의 한 칸에 맞춰 변형하지 않고, 공개 점수열과 표준 판정규칙에서 재계산한 `HVI=NO`를 유지했다.

### C. Tibon Czopp et al. 공개 사례와 정오표

- 반응 수: 21
- 검증 범위: 점수열, 운동 방향, 음영, Blends, Sum6, WSum6
- 논문: [The amnestic syndrome: applying the Rorschach Inkblot method for differential diagnosis](https://pubmed.ncbi.nlm.nih.gov/23985019/)
- 정오표: [Corrigendum](https://www.tandfonline.com/doi/pdf/10.1080/13554794.2014.910345)

대표 일치값: `R=21`, `Lambda=.24`, `Blends=9`, `a:p=7:7`, `Sum6=9`, `WSum6=28`.

정오표는 `FC:CF+C=2:5`라고 적고, 공개 점수열에는 `FC=2`, `CF=4`, `C=0`, `Cn=1`이 있다. 당시 감사는 WSumC의 Cn 제외 규칙을 화면 비율에도 잘못 적용해 `2:4`를 유지했으나, 후속 재감사에서 관례적 화면 비율은 Cn을 포함한 `2:5`가 맞음을 확인했다. v2.2.0의 Cn 누락은 v2.2.1 런타임에서 교정됐고, v2.2.2에서 별도 경계 테스트로 고정했다.

### D. 초기 v1 개발에 참고한 공개 배포 2019 Excel workbook

- 반응 수: 16
- 검증 범위: v1 GAS 이전 원수식과 v2 TypeScript 마이그레이션 회귀
- 배포 위치: [Naver Blog 게시물](https://blog.naver.com/jin_k84/221539279596). workbook 내부 제작 표기는 `[Scoring Program] _by. Ju-Ri`이며 실명을 추정하거나 원본 파일을 재배포하지 않음

대표 일치값: `R=16`, `Lambda=.33`, `EB=4:1`, `EA=5`, `EBPer=4`, `Zf=13`, `ZSum=34`, `ZEst=41.5`, `Zd=-7.5`, `a:p=9:2`, `3r+(2)/R=.3125`, `CDI=4 Positive`.

Excel은 매우 중요한 독립 기준이지만 무조건 정답으로 복사하지 않았다. 예를 들어 오래된 `m'a`, `m'p`, `m'a-p` 표기는 현행 앱의 `ma`, `mp`, `ma-p`로 정규화했고, EBPer 표시 방식처럼 이후 표준 기준과 맞지 않는 흔적은 현행 코드에 계승하지 않았다.

## 문화권 자료를 계산 정답지와 분리한 이유

같은 부호화 값이 입력되면 구조요약 산식 자체는 한국, 일본, 미국에서 달라지지 않는다. 문화권 차이는 주로 어떤 반응을 FQ 또는 P로 부호화하는지, 규준과 해석을 어떻게 적용하는지에 영향을 준다. 따라서 국가별 집단 평균을 개인 반응열의 계산 정답처럼 사용하면 오히려 잘못된 검증이 된다.

- 한국 아동 1,049명의 22,067개 반응 연구는 비교 가능한 63개 변수 중 53개가 미국 아동 규준과 유의하게 달랐다고 보고했다: [한국형 아동 로르샤하 종합체계 탐색 연구](https://www.accesson.kr/kjcp/v.28/4/961/16620)
- 일본 정상 성인 여성 102명의 Lambda 연구는 Lambda를 다른 구조요약 변수와 함께 해석해야 한다고 보고했다: [The Lambda in the Rorschach comprehensive system](https://www.jstage.jst.go.jp/article/jjpsy1926/73/6/73_6_502/_article)
- 일본 섭식장애 입원환자 50명과 일본 건강집단 240명을 비교한 CS 연구도 문화권 내부 비교자료를 사용했다: [Japanese eating-disorder CS study](https://www.jstage.jst.go.jp/article/jjpm/46/3/46_KJ00004075679/_article/-char/ja/)

이 원칙에 따라 계산기는 완전한 점수열과 기대 출력이 있는 네 사례로 검증하고, 문화·언어 차이는 한국어·일본어·영어 각 3개씩 총 9개의 GPT-5.5 부호화·해석 시나리오로 별도 검증했다.

## 남아 있는 한계

1. 이 감사는 독립 사례와 경계값을 크게 늘렸지만 모든 가능한 반응 조합을 수학적으로 증명한 것은 아니다.
2. 출판 사례의 표에도 오류가 있을 수 있으므로 점수열, 표, 판정규칙이 충돌하면 자동으로 한쪽을 믿지 않고 차이를 기록해야 한다.
3. 계산기는 현재 나이를 입력받지 않으므로 `S-CON`의 12개 조건 점수는 나이와 무관하게 산출된다. 양성·음성 해석은 만 15세 이상에서만 적용하며, 만 14세 이하에서는 계산값을 판정 결론으로 사용하지 않는다. v2.2.0은 이 제한을 5개 언어 코퍼스, 고정 AI 하네스, 계산 결과 화면의 안내 문구에 일치시켰다. 다만 계산기가 연령을 직접 입력받아 판정을 숨기거나 보류하지는 않으므로 연령 인식 자체는 후속 제품 판단이 필요하다.
4. 짧거나 불완전한 반응 기록의 구조요약 숫자가 계산되더라도 임상적으로 유효한 기록이라는 뜻은 아니다.

## 릴리즈 판단

발견된 일곱 결함은 v2.2.0 로컬 코드에서 수정되었고 회귀 테스트로 고정했다. 따라서 수정 전 v2.1.10의 계산 결과를 “완전 무결”이라고 표현해서는 안 된다. 반면 현재 감사 근거에서는 계산 체계 전체를 다시 작성해야 할 광범위한 붕괴도 발견되지 않았다.

v2.2.0은 수정된 코드, 네 개의 독립 계산 정답지, 경계값 테스트, 전체 회귀 테스트를 모두 통과한 뒤에만 릴리즈한다.
