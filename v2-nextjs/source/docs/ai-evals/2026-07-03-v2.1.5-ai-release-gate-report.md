# v2.1.5 AI Release Gate Report

Date: 2026-07-03

Status: local implementation pass, release candidate

## Goal

v2.1.5 continues the v2.1.x AI quality work without changing UI/UX.

The goal is to make AI release readiness repeatable:

- one command for the core local AI release gate
- the same gate wired into GitHub Actions
- privacy-safe reports that record pass/fail state without raw prompts or raw model answers
- clearer separation between AI quality gates and live API spending

## What Changed

### AI Release Gate Runner

Added:

```bash
npm run ai:release-gate
```

The command runs:

1. `npm run ai:evaluate-contracts`
2. `npm run ai:evaluate-artifacts`
3. `npm run ai:evaluate-human-records -- --require-pass`
4. `npm run security:secrets`

The runner stops on the first required failure and returns a non-zero exit code.

It also supports:

```bash
npm run ai:release-gate -- --dry-run
npm run ai:release-gate -- --json
npm run ai:release-gate -- --markdown docs/ai-evals/example-gate-report.md
```

### CI Wiring

The GitHub `security-check` workflow now runs:

```bash
npm run ai:release-gate
npm run security:check
```

That means future `main` pushes check the AI harness contracts, saved eval artifacts, human review records, secret scan, and dependency audit before the release is considered clean.

## Privacy Boundary

The release gate records command names, step status, exit codes, timing, and optional markdown/json summaries only.

It does not store:

- API keys
- raw prompts
- raw model answers
- raw chat messages
- private assessment payloads

## Remaining Gaps

- The release gate does not run live OpenAI calls by default. This is intentional so normal release checks do not spend API budget.
- Live eval batches should still be run explicitly when we want fresh model evidence.
- The moderate `js-yaml` advisory should be reviewed separately from the AI quality line.
