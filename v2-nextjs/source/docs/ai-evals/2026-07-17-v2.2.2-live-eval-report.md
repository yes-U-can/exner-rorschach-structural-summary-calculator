## 먼저 읽을 결론

이번 검사는 선택 기능인 AI 도우미가 Cn이 들어가는 계산과 들어가지 않는 계산을 정확히 구분해 설명하는지 확인하기 위해 진행했습니다. 실제 GPT-5.5 API를 호출해 답변까지 확인하는 방식을 이 보고서에서는 `live eval`이라고 부릅니다.

API 키 없이 사용하는 구조요약 계산기는 AI와 관계없이 같은 계산식을 사용합니다. 또한 v2.2.1의 `FC:CF+C` 오른쪽 값은 이미 Cn을 포함해 올바르게 계산됐으므로, **완성된 프로토콜은 이 이유만으로 다시 계산할 필요가 없습니다.**

마지막 검사에서는 한국어, 영어, 일본어, 스페인어, 포르투갈어를 각각 두 번씩 질문해 **10/10 통과**했습니다. 해석 도우미의 집중 확인도 **3/3 통과**했고, 기존의 대표적인 다섯 언어 질문도 **5/5 통과**했습니다.

## AI 도우미가 구분해야 하는 네 가지 계산

`FC=2`, `CF=4`, `C=0`, `Cn=1`인 완성된 구조요약을 예로 들면 다음과 같습니다.

| 사용 위치 | 올바른 계산 | 예시 결과 |
| --- | --- | --- |
| 화면의 관례적 라벨 `FC:CF+C` | `FC : (CF + C + Cn)` | `2:5` |
| WSumC | `0.5*FC + 1.0*CF + 1.5*C` | `5.0` |
| S-CON 7번 기준 | `CF + C > FC` | `4 > 2` |
| Color-Shading의 색채 결정인 | `FC`, `CF`, `C`만 포함 | Cn 제외 |

Cn은 첫 번째 화면 비율에는 들어가지만 나머지 세 계산에는 들어가지 않습니다. 비슷해 보이는 색채 합계를 하나로 묶어 쓰면 다른 계산에 Cn이 잘못 섞일 수 있으므로, 계산 코드와 AI 설명에서 네 범위를 각각 나누었습니다.

### 코딩 도우미의 문서 찾기

코딩 도우미가 Cn 부호 문서만 찾으면 Lower Section의 계산 범위를 놓칠 수 있습니다. 그래서 Cn을 직접 묻는 질문에는 부호 문서와 연결된 Lower Section 설명도 함께 찾도록 했습니다. 한국어와 일본어처럼 `Cn` 뒤에 조사가 바로 붙는 문장도 부호로 인식합니다.

### 답변 확인 기준

정답 문장을 한 글자도 바꾸지 못하게 요구하지는 않습니다. 대신 다음 항목은 분명하게 확인합니다.

1. `2:5`, `CF+C+Cn`처럼 계산을 나타내는 표기는 정확해야 합니다.
2. `12:5`나 `2:55` 안의 일부 숫자를 `2:5`로 잘못 인정하지 않습니다.
3. WSumC, S-CON, Color-Shading에서 Cn을 제외한다는 설명을 각각 확인합니다.
4. 자연스러운 어순과 번역 차이는 허용하되, 네 계산 중 하나라도 빠지거나 반대로 설명하면 통과시키지 않습니다.
5. 일본어의 띄어쓰기 없는 문장이나 긴 답변도 같은 기준으로 확인합니다.

즉, 특정 모범 문장을 외웠는지가 아니라 필요한 계산 관계를 정확히 설명했는지를 확인합니다.
