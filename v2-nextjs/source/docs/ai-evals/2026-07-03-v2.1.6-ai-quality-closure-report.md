# v2.1.6 AI Quality Closure Report

Date: 2026-07-03

Status: local implementation pass, release candidate

## Goal

v2.1.6 closes the initial v2.1.x assistant-harness and release-gate scope without changing UI/UX.

The goal is to make the OpenAI assistant layer more release-ready before v2.2.0 moves to mobile user experience work.

## Retrospective Scope Note (2026-07-13)

The original release wording described the entire v2.1.x AI quality line as closed. Later user testing, corpus recalibration, and independent retrieval audits exposed additional, separately bounded work. The final sequence should therefore read as follows: v2.1.6 closed the initial harness and release-gate foundation; v2.1.7 formalized public documentation governance; v2.1.8 recalibrated the corpus and vectors; v2.1.9 hardened hybrid retrieval; and v2.1.10 closed the confirmed follow-up audit defects.

This note narrows the earlier closure claim without changing the historical v2.1.6 implementation or verification results.

## What Changed

### Full Runtime Retrieval Gate

Added:

```bash
npm run docs:evaluate-rag:all
```

The command generates the reference corpus once, then runs curated retrieval evals for every runtime locale:

- KO: 101 queries
- EN: 61 queries
- JA: 61 queries
- ES: 81 queries
- PT: 61 queries

This is now part of:

```bash
npm run ai:release-gate
```

### PT Retrieval Eval Fix

Fixed a release-gate blind spot where `docs:evaluate-rag:pt` could fall back to the KO locale path because the eval locale switch did not include `pt`.

After enabling the real PT path, the PT eval exposed retrieval regressions. The retrieval intent layer was updated so Portuguese coding, interpretation, graph-navigation, and code-like symbol questions are ranked correctly.

Examples covered by the PT gate include:

- `DQ+` versus `DQo` / `DQv/+`
- `Cartao V`
- `M-`
- `3r+(2)/R`
- `FM`
- `COP` coding versus interpretation contrast

### Stronger AI Release Gate

The default AI release gate now runs:

1. `npm run docs:evaluate-rag:all`
2. `npm run ai:evaluate-contracts`
3. `npm run ai:evaluate-artifacts`
4. `npm run ai:evaluate-human-records -- --require-pass`
5. `npm run security:secrets`
6. `npm run security:audit`

The gate remains privacy-safe. It records command names, pass/fail status, exit codes, and timing only.

### Dependency Audit Cleanup

Updated the locked `js-yaml` versions used under `gray-matter` and eslint tooling.

The dependency audit threshold was tightened from high severity to moderate severity:

```bash
npm audit --omit=dev --audit-level=moderate
npm audit --audit-level=moderate
```

Current result: zero reported vulnerabilities.

## Privacy Boundary

No API key was committed or written to eval artifacts.

The release gate does not store:

- raw prompts
- raw model answers
- raw chat messages
- private assessment payloads

Live OpenAI evals remain explicit commands so normal release checks do not accidentally spend API budget.

## Remaining Gaps

- The saved live eval artifacts remain KO/EN focused. They are useful engineering evidence, but not a substitute for expert clinical review.
- Human review records currently cover representative examples, not a full clinical validation set.
- At the time of v2.1.6, further AI work was expected to move into a new version line. Later evidence justified the bounded v2.1.7-v2.1.10 follow-up sequence described in the retrospective scope note above.
