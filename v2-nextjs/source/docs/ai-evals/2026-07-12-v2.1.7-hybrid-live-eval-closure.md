## Defects Found By Live Eval

### Assumed Missing Values

The first multi-turn run showed that GPT-5.5 could comply when asked to assume missing Lambda and DEPI values and draft a case conclusion. The interpretation guardrail now forbids substituting assumed, invented, hypothetical, or user-suggested values for missing case data. It requires actual observed values before producing case-specific or report-ready conclusions.

### Stale Multi-turn Row Context

The first live test simulated a user selecting a new row while keeping the system prompt focused on row 1. The test now mirrors the application request lifecycle: each turn rebuilds the system prompt, row context, retrieval query, and hybrid evidence from the current focus row while preserving the conversation history.

The coding prompt also now states that every row has independent evidence and that candidates must not be copied from a prior row merely because the content label looks similar.

### Brittle String Scoring

The evaluator previously missed safe equivalents such as `without the actual values`, smart-apostrophe contractions, and phrases with short inserted modifiers. Required-signal matching now normalizes Unicode typography and supports bounded ordered-token gaps. A generic `has depression` forbidden substring was removed because correct boundary answers often need to repeat that wording while rejecting it.

### Locale Contamination

An English live response occasionally switched into Korean because Korean-only formatting examples were included in every locale's system prompt. Interpretation prompts are now assembled per locale. Korean examples are injected only for `ko`, and the eval contract reports substantial cross-script leakage as `locale_mismatch`.
