Date: 2026-07-13
Scope: AI/RAG backend, migration replay, eval evidence, and release gates
UI/UX change: None, apart from version metadata

## Summary

After v2.1.9 was deployed, Claude Fable 5 performed a third read-only independent audit. It rated the release `PASS WITH NON-BLOCKING FINDINGS` and `Release blocker: NO`, while identifying four P1 follow-up candidates. GPT-5.6 Sol / Codex then reproduced every candidate against the local source and an isolated pgvector database before changing code.

All four candidates were confirmed. v2.1.10 closes them instead of carrying them into the mobile-focused v2.2.x series. The patch does not change the calculator UI, clinical scope, GPT-5.5 model policy, BYOK boundary, or HITL rules.

## Confirmed Findings And Fixes

| Finding | Before | v2.1.10 fix |
| --- | --- | --- |
| Lexical fallback item/trace mismatch | Broad fallback `items` could still contain scoring routes even when the independently ranked trace filtered them | Rank lexical fallback once and return its aligned `items` and `trace`; broad queries are constrained to interpretation routes before fallback |
| Japanese attached code symbols | `FQ+の...` ranked `FQ-` first and `v/+の...` missed the DQ route because `Intl.Segmenter` detached the symbol | Extract complete Latin/code spans before Japanese segmentation; cover `FQ+`, `FQ-`, `v/+`, and `3r+(2)/R` with attached Japanese text |
| Broad-intent expression gap | Natural variants such as “results as a whole” and `결과 요약을 전체적으로` were not classified as broad and returned scoring routes | Expand natural broad expressions in all five locales and apply an explicit interpretation-only route boundary rather than relying only on the top lexical anchor |
| Fresh migration replay failure | A historical uniqueness migration ran before the embedding table existed, causing `P3018 / 42P01` on a clean database | Add an idempotent preparatory migration before the historical repair; keep already-applied migration files unchanged |

The migration decision follows Prisma's guidance not to edit applied migrations. The preparatory step is safe both when replayed from empty and when applied late to an existing final schema. See [Prisma migration history guidance](https://www.prisma.io/docs/orm/prisma-migrate/understanding-prisma-migrate/migration-histories) and [migrate deploy](https://www.prisma.io/docs/cli/migrate/deploy).

## Additional Audit Hardening

- Hybrid mode now reports `lexical` when no usable vector hit survives, instead of reporting `hybrid` with zero vector hits.
- Keyless lexical/vector copies use a stable title-and-content merge key rather than an index-dependent key.
- Live evals treat an empty or whitespace-only `OPENAI_API_KEY` as absent.
- Hybrid, batch, and route evidence now records app version, source commit, and dirty-tree state.
- The production-like route eval now writes a sanitized JSONL artifact. Raw model output remains in memory and is not persisted.
- The local ignored production environment file was aligned with the verified maintenance database target after an old empty target was detected. No secret value was committed or printed.
- GitHub CI now replays the complete Prisma history against an empty pgvector/PostgreSQL service and verifies OpenAI-only enum state, the locale-aware unique index, and late-application data preservation.

## Corrections To v2.1.9 Wording

- “365 curated queries” means `360` curated retrieval queries plus `5` schema/contract checks.
- Japanese symbol preservation in v2.1.9 covered separated forms; attached suffix forms such as `FQ+の...` were incomplete.
- Broad contamination `0%` was true for the ten development expressions then present, not for every possible natural paraphrase.
- Exact-route and metadata bonuses are bounded rerank signals, but a stacked exact match can exceed one single-source RRF contribution. It did not restore the old lexical-score domination failure.

These corrections are preserved as part of the audit trail rather than hidden by the later patch.

## Remaining Limits

- The 30-query hybrid set is a development challenge, not a blind held-out benchmark.
- Broad intent still uses explicit multilingual language rules; the hard route boundary reduces the impact of a missed expression, but cannot prove coverage of every paraphrase.
- The production-like route eval still covers one synthetic interpretation scenario.
- Automated contracts verify engineering behavior, not clinical correctness.
- Human expert review remains required for clinical usefulness, multilingual nuance, and real assessment interpretation.
