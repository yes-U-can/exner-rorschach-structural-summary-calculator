# [2026-02-15] v2.0.0 메이저 패치

v2.0.0은 Google Apps Script로 만든 v1.4.1을 Next.js 기반 웹앱으로 다시 설계한 첫 번째 버전 2 릴리즈입니다. v1의 핵심이었던 로샤 Exner(CS) 체계 구조요약 계산 흐름은 유지하되, 더 안정적인 웹앱 구조, 다국어 UI, 참조 문서 검색, 그리고 BYOK 방식의 AI 보조 기능을 추가했습니다.

이번 버전의 가장 큰 변화는 AI 기능을 서비스 내부 과금이나 플랫폼 제공 모델이 아니라 사용자의 API 키로만 동작하도록 만든 점입니다. BYOK는 Bring Your Own Key의 약자로, 사용자가 본인의 OpenAI 또는 Google API 키를 직접 입력해 AI 기능을 사용하는 방식입니다. 이 방식은 서비스 운영자가 사용자의 AI 대화 비용을 대신 결제하거나 API 키를 서버 DB에 보관하지 않아도 되게 하며, 사용자가 자신의 AI 사용 범위와 비용을 직접 관리할 수 있게 합니다.

## v1.4.1에서 유지한 것

v1.4.1의 핵심 목적은 그대로 유지했습니다. 사용자는 로샤 반응을 행 단위로 입력하고, 카드, 반응 위치, 발달질, 결정인, 형태질, 반응 내용, 특수점수 등 구조요약 계산에 필요한 값을 정리한 뒤 결과를 계산할 수 있습니다.

v1에서 제공하던 구조요약 계산 흐름, 결과 확인, CSV 내보내기, 다국어 지원 방향도 v2에서 이어받았습니다. 다만 v2에서는 입력 화면과 결과 화면을 더 큰 웹앱 구조 안에서 다시 구성했고, 모바일과 데스크톱에서 유지보수하기 쉬운 컴포넌트 구조로 정리했습니다.

## 새 웹앱 구조

v2는 Next.js App Router 기반으로 작성되었습니다. v1 GAS 버전이 단일 Google Apps Script 웹앱에 가까웠다면, v2는 라우팅, 서버 API, 클라이언트 UI, 참조 문서 데이터, AI 요청 처리를 분리한 구조입니다.

주요 화면은 채점 화면, 해석 도우미, 참조 문서, 계정 관리, 버전 아카이브로 나뉩니다. 사용자는 로그인하지 않아도 기본 계산 기능과 참조 문서 검색을 사용할 수 있고, AI 기능을 쓰려면 로그인 후 이번 세션에서 사용할 API 키를 연결해야 합니다.

## BYOK AI 세션

v2의 AI 기능은 BYOK 전용으로 동작합니다. 플랫폼 크레딧, 결제, 월정액, 스토어, 플랫폼 제공 AI 모델은 포함하지 않았습니다. 사용자는 OpenAI 또는 Google 중 하나의 API 키만 연결할 수 있으며, 두 제공자를 동시에 저장하도록 설계하지 않았습니다.

API 키는 서버 DB에 저장하지 않고 브라우저 세션에서만 사용합니다. 로그아웃하거나 세션이 만료되면 API 키도 더 이상 유지되지 않습니다. 사용자가 API 키를 입력할 때에는 OpenAI 키와 Google 키의 기본 형식을 구분하여, 잘못된 탭에 다른 제공자의 키를 넣는 실수를 막도록 했습니다.

모델은 사용자가 직접 고르지 않습니다. 웹앱은 출시 후 일정 기간이 지나 안정성이 확인된 모델 중에서 서비스가 정한 최신 모델을 자동으로 사용합니다. v2.0.0 기준 OpenAI는 GPT-5.4, Google은 Gemini 2.5 Pro를 기본 모델로 둡니다.

AI 제공자가 요청을 거절하는 경우에도 오류를 하나로 뭉개지 않도록 정리했습니다. API 키가 잘못된 경우, 결제 또는 사용량 한도 문제가 있는 경우, 해당 API 키로 선택된 모델을 사용할 수 없는 경우를 구분해 사용자에게 안내합니다.

## 코딩 도우미

채점 화면의 AI는 코딩 도우미로 정리했습니다. 여기서 말하는 코딩은 프로그래밍이 아니라, 로샤 반응을 Exner(CS) 체계의 기호와 범주로 부호화하는 과정을 뜻합니다.

코딩 도우미는 Ctrl/Cmd+J 단축키로 열 수 있습니다. 사용자가 현재 채점 화면에 입력한 전체 시트가 기본 맥락으로 전달되며, 사용자가 특정 행을 선택한 경우에는 그 행이 더 중요한 맥락으로 강조됩니다. 아무 행도 선택하지 않은 상태에서는 특정 중심 행 없이 전체 시트 맥락만 사용합니다.

코딩 도우미는 자동으로 답을 확정하는 기능이 아니라, 사용자가 판단할 수 있도록 후보와 근거를 제안하는 대화형 보조 도구입니다. 반응의 위치, 결정인, 형태질, 내용 범주 등을 검토할 때 필요한 질문을 던지고, 참조 문서와 입력 맥락을 바탕으로 가능한 후보를 설명하도록 설계했습니다.

자동 채우기 버튼은 v2.0.0 최종 공개 범위에서 제외했습니다. 부호화는 임상가가 반응 맥락을 검토하며 판단해야 하는 작업이므로, 한 번의 버튼 클릭으로 행을 채우는 방식은 HITL 설계 원칙과 맞지 않는다고 판단했습니다.

## 해석 도우미

해석 도우미는 구조요약 수치 CSV를 바탕으로 해석 대화를 이어가기 위한 AI 화면입니다. 사용자는 구조요약 수치가 담긴 CSV 파일을 첨부하고, 연령, 성별, 관찰 맥락, 임상가가 이미 생각한 가설이나 질문을 함께 적어 AI에게 검토를 요청할 수 있습니다.

해석 도우미는 구조요약 결과의 패턴을 설명하고, 사용자가 세운 가설을 점검하는 보조 역할을 합니다. 공식 진단이나 최종 해석을 대신하지 않으며, 임상가의 판단을 대체하지 않도록 안내 문구와 시스템 지침을 구성했습니다.

## 참조 문서

v2는 웹앱이 참고하는 문서 코퍼스를 함께 제공합니다. 여기서 말하는 참조 문서는 구조요약 계산과 부호화, 해석 과정에서 웹앱과 AI가 근거로 삼는 짧은 설명 문서입니다. 사용자는 참조 문서 화면에서 이 문서를 직접 검색할 수 있습니다.

참조 문서는 서울임상심리연구소와 모오가 함께 제작하고 정리한 자료를 바탕으로 구성했습니다. AI가 어떤 지식을 근거로 답변하는지 사용자가 확인할 수 있도록 공개하는 것이 목표입니다. 이는 AI 기능을 블랙박스로 두지 않고, 사용자가 근거를 확인하며 판단할 수 있게 하려는 설계 원칙과 연결됩니다.

참조 문서 URL에서는 `+`, `-`, `v/+` 같은 부호 코드가 경로에서 깨지지 않도록 안전한 slug를 사용했습니다. 내부 문서 경로와 브라우저 URL을 분리하여, 관련 문서 링크와 상단 이동 위젯에서 404가 발생하지 않도록 정리했습니다.

## 인간 중심 AI와 HITL 원칙

v2는 AI가 전문가를 대신해 판단하는 시스템이 아니라, 임상심리사와 수련생이 더 명확한 근거 위에서 검토하고 판단하도록 돕는 시스템을 목표로 합니다. 그래서 AI 기능은 두 가지로만 남겼습니다. 채점 화면에서는 코딩 도우미, 별도 채팅 화면에서는 해석 도우미입니다.

AI 답변은 참고와 보조를 위한 것이며, 최종 부호화와 해석 책임은 사용자에게 있습니다. 이 방향은 Human-in-the-Loop, 즉 사람이 판단 과정 안에 남아 있어야 한다는 원칙과 연결됩니다.

## 보안과 개인정보

v2.0.0에서는 사용자 API 키와 AI 대화 기록을 서버 DB에 장기 저장하지 않는 방향으로 정리했습니다. API 키는 이번 로그인 세션에서만 사용하고, 로그아웃 또는 세션 만료 시 사라지도록 설계했습니다.

서비스는 플랫폼 크레딧, 결제, 월정액, 사용자 스킬북 스토어, 공개 스토어 moderation 같은 기능을 포함하지 않습니다. 이 기능들은 개발 과정에서 검토되었으나, v2.0.0의 실제 공개 범위에서는 제거했습니다.

## UI/UX 변경

v2는 전체 UI를 웹앱 형태로 다시 구성했습니다. 상단 헤더, 언어 선택 드롭다운, 라이트/다크 모드, 계정 관리 화면, 버전 아카이브, 참조 문서 검색 화면을 정리했습니다.

채점 화면에서는 행 선택, 행 추가/삭제, 단축키, undo/redo 흐름을 정비했습니다. 선택된 행은 지속 강조 상태로 표시되며, 선택 상태에 따라 코딩 도우미가 받는 맥락도 달라집니다.

해석 도우미 화면에는 CSV 첨부 버튼과 드래그앤드롭 첨부를 추가했습니다. 단순히 텍스트를 붙여넣는 방식뿐 아니라, 구조요약 수치 CSV를 첨부하고 그 내용을 AI 대화 맥락으로 사용할 수 있습니다. 첨부 파일은 한 번에 하나만 받을 수 있도록 제한하여, 여러 자료가 섞여 AI가 잘못된 맥락을 읽는 일을 줄였습니다.

## 다국어 지원

v2는 한국어, 영어, 일본어, 스페인어, 포르투갈어 UI를 지원합니다. 언어 버튼을 개별적으로 나열하던 방식에서 드롭다운 선택 방식으로 정리하여, 화면 크기와 언어별 문자열 길이에 따른 레이아웃 흔들림을 줄였습니다.

문자 깨짐을 막기 위해 locale 파일을 검사하는 `i18n:audit` 스크립트를 추가했습니다. `??`, 깨진 대체 문자, 대표적인 mojibake 패턴이 배포 전에 감지되도록 했습니다.

## 제거한 기능

v2.0.0 최종 공개 범위에서는 플랫폼 제공 AI, 크레딧, 결제, 월정액, 사용자 스킬북, 스킬북 스토어, 해석 노트 빌더, 관리자 스토어 moderation, credit metrics 기능을 제거했습니다.

이 결정은 제품을 더 작고 명확하게 만들기 위한 선택입니다. 이번 버전의 핵심은 구조요약 계산, 참조 문서, BYOK 기반 코딩 도우미와 해석 도우미입니다.

## 공개 소스 범위

이 공개 저장소에는 v2를 이해하고 검토하는 데 필요한 소스코드, 참조 문서 코퍼스, 번역 파일, 테스트, 빌드 설정을 포함합니다. 반대로 실제 운영 환경변수, Vercel 프로젝트 설정, 로컬 로그, 캐시, node_modules, 과거 실험적 운영 문서, 비공개 작업 노트는 포함하지 않습니다.

공개 목적은 사용자가 이 웹앱의 작동 방식과 설계 의도를 확인할 수 있게 하는 것입니다. 운영 비밀이나 불필요한 개발 흔적까지 전시하는 것이 목적은 아닙니다.
