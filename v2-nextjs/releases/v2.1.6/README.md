# [2026-07-04] v2.1.6 버그 패치

## 요약

v2.1.6은 v2.1.2부터 마련한 AI 답변 규칙과 릴리즈 전 자동 검사를 하나의 반복 가능한 기준으로 묶은 패치입니다.

앱 UI/UX는 변경하지 않았습니다. 다섯 언어의 참조 문서 검색, AI 답변 기준, 저장 기록, 사람 검토, 비밀정보와 외부 프로그램 취약점을 한 번에 확인하도록 정리했습니다.

## 후속 기록 (2026-07-13)

처음 쓴 패치노트에는 v2.1.6으로 v2.1.x의 AI 품질 작업 전체를 마무리한 것처럼 적었습니다. 이후 실제 사용과 재검토에서 참조 문서와 검색 방식에 별도의 개선이 더 필요하다는 점을 확인했습니다.

따라서 v2.1.6에서 완료한 범위는 **초기 답변 규칙과 릴리즈 전 자동 검사**까지로 바로잡습니다. 공개 문서 작성 원칙은 v2.1.7에서, 참조 문서 갱신과 다국어 검색 개선은 v2.1.8-v2.1.10에서 이어서 다뤘습니다. 이 정정은 v2.1.6 당시의 구현과 검사 결과를 바꾸지 않습니다.

## 핵심 요약

- 다섯 언어의 대표 질문을 한 번에 실행해 관련 참조 문서를 찾는지 확인할 수 있게 했습니다.
- 포르투갈어 검사가 한국어 질문으로 잘못 대체될 수 있던 문제를 고쳐 실제 포르투갈어 질문을 사용하도록 했습니다.
- 포르투갈어의 코딩, 해석, 연결 문서 찾기, 짧은 부호 질문을 더 정확히 구분하도록 검색을 개선했습니다.
- 릴리즈 전 종합 점검에 다섯 언어 참조 문서 검색과 외부 프로그램 취약점 검사를 포함했습니다.
- 보안 권고를 차단하는 기준을 높은 심각도에서 중간 심각도까지 넓혔습니다.
- `js-yaml` 버전을 갱신해 알려진 취약점이 0건이 되도록 했습니다.
- v2.1.6 검사 결과는 `source/docs/ai-evals/`에 추가했습니다.

## 어떻게 확인했나요?

다섯 언어 참조 문서 검색, AI 답변 기준, 저장 기록, 사람 검토, 비밀정보, 외부 프로그램 취약점의 여섯 단계를 모두 통과했습니다.

<details>
<summary><strong>개발자가 같은 검사를 다시 실행할 때 사용하는 명령과 세부 단계</strong></summary>

```bash
npm run ai:release-gate -- --markdown docs/ai-evals/2026-07-03-v2.1.6-ai-quality-closure-gate.md

```

AI release gate는 다음 6개 필수 단계를 모두 통과했습니다.

1. `npm run docs:evaluate-rag:all`
2. `npm run ai:evaluate-contracts`
3. `npm run ai:evaluate-artifacts`
4. `npm run ai:evaluate-human-records -- --require-pass`
5. `npm run security:secrets`
6. `npm run security:audit`

</details>

## 공개 범위와 보안 경계

이 공개 아카이브에는 배포용 비밀정보, 로컬 환경 파일, API 키, 실행 기록, 비공개 데이터베이스, Vercel 프로젝트 설정을 포함하지 않았습니다.

저장된 AI 검사 결과에는 검사 항목 번호, 완료 상태, 발견된 문제 종류, 사용량, 답변 길이, 추정 비용처럼 공개 가능한 요약 정보만 기록했습니다. 질문·답변 원문, 비공개 평가 자료, API 키는 저장하지 않았습니다.
