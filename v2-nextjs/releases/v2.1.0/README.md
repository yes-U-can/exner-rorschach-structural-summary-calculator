# [2026-06-22] v2.1.0 마이너 패치

v2.1.0은 지원 브라우저에서 웹앱을 설치형 앱처럼 열 수 있게 하고, 초기 개발에 도움을 준 RorScore에 대한 감사 기록을 남기며, AI가 코딩 질문에 알맞은 참조 문서를 먼저 찾도록 개선한 패치입니다.

## 핵심 요약

- Chrome/Edge 등 지원 브라우저에서 웹앱을 설치형 앱처럼 열 수 있도록, 앱 이름·아이콘·시작 주소를 브라우저에 알려주는 설치 정보 파일을 추가했습니다.
- 민감한 평가 데이터나 AI 응답을 기기에 별도로 저장하지 않도록 오프라인 저장, 푸시 알림, 백그라운드 동기화 기능은 추가하지 않았습니다.
- 설치 정보에 192px/512px 앱 아이콘을 명시하고, 현재 웹앱만 설치 대상으로 나타나는지 확인했습니다.
- `DQ+` 같은 코딩 질문에서 AI 참조문서 검색이 비교 대상으로 언급된 해석 문서보다 해당 채점 입력 문서를 먼저 찾도록 랭킹을 보정했습니다.
- 초기 Google Apps Script 구현 시기에 Jeremy Leader의 RorScore가 중요한 학습 참고 자료였음을 `ACKNOWLEDGEMENTS.md`에 기록했습니다.

## 영향받은 흐름

설치 기능은 앱의 저장 방식이나 API 키 보안 모델을 바꾸지 않습니다. 사용자가 본인의 API 키를 연결하는 BYOK(Bring Your Own Key) 키는 기존과 동일하게 서버 데이터베이스나 브라우저의 일반 저장소에 남기지 않고, 24시간짜리 HttpOnly 쿠키에 암호화해 보관합니다.

이번 버전의 설치 지원은 앱 이름, 시작 주소, 표시 방식, 아이콘 같은 정보만 브라우저에 알려주는 수준입니다. 민감한 평가 데이터나 AI 응답을 오프라인에 저장하는 기능은 도입하지 않았습니다.

AI 참조문서 검색 보정은 계산 결과에는 영향을 주지 않습니다. 다만 코딩 도우미가 "DQ+는 언제 DQo나 DQv/+가 아니라 DQ+로 코딩해야 하나?" 같은 질문을 받았을 때, 해석 문서보다 채점 입력 문서를 우선 참고하도록 품질을 개선했습니다.

## 감사 기록

v2.1.0에는 RorScore에 대한 뒤늦은 감사 기록도 포함됩니다. 이 프로젝트는 RorScore의 코드를 복사하거나 포함하지 않지만, 초기 GAS 구현을 시작하던 시기에 RorScore는 로샤 구조요약 계산 로직을 코드로 표현할 수 있다는 것을 보여준 중요한 학습 참고 자료였습니다.

자세한 내용은 루트의 `ACKNOWLEDGEMENTS.md`에서 확인할 수 있습니다.

## 어떻게 확인했나요?

v2.1.0 배포 전 다음 검사를 통과했습니다.

- 코드 형식과 다섯 언어 문구 검사
- 단위/통합 테스트 68건
- 5개 언어 참조 문서 검색 검사
- 참조 문서가 실제 검색에 사용될 준비가 됐는지 확인
- 배포용 페이지 생성
- 심각도가 높은 알려진 취약점과 비밀정보 점검
- 브라우저가 앱 설치 정보와 아이콘을 정상적으로 읽는지 확인

GitHub `main` 반영 후 Vercel 운영 배포가 완료되고 실제 웹앱의 푸터와 버전 아카이브가 갱신됐는지 확인했습니다.
