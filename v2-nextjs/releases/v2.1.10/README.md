# [2026-07-13] v2.1.10 버그 패치

## 먼저 확인하실 점

v2.1.10은 v2.1.9 이후에도 남아 있던 참조 문서 검색과 새 데이터베이스 설치 문제 네 가지를 수정한 패치입니다.

앱 UI/UX와 계산식은 변경하지 않았습니다. GPT-5.5 고정 모델, 본인의 API 키 사용, 임상가 최종 판단 원칙도 그대로 유지했습니다. 같은 검색 결과를 일관되게 기록하고, 일본어 문장에 붙은 로샤 부호를 보존하며, 넓은 해석 질문에는 해석 문서만 사용하고, 새 데이터베이스도 오류 없이 준비되도록 했습니다.

## 핵심 요약

- 넓은 질문에서 실제로 반환한 문서와 검사 기록에 적힌 문서가 달라질 수 있던 문제를 하나의 정렬 결과로 통합했습니다.
- `FQ+の...`, `v/+の...`, `3r+(2)/Rの...`처럼 로샤 코드에 일본어가 붙은 질문에서도 완전한 코드 토큰을 먼저 보존하도록 수정했습니다.
- 다섯 언어의 자연스러운 전체 해석 질문을 보강하고, 넓은 질문에는 해석 문서만 반환하도록 제한했습니다.
- 새 PostgreSQL 데이터베이스에 29개 변경 이력을 처음부터 적용할 때, 필요한 표가 만들어지기 전에 이를 참조해 설치가 중단되던 문제를 해결했습니다. 이미 배포된 과거 변경 파일은 수정하지 않았습니다.
- 의미 검색 결과가 하나도 없을 때 검색 방식을 혼합 검색으로 잘못 기록하지 않고 단어 검색으로 표시하도록 했습니다.
- 검색 결과에 내부 번호가 없어도 문서 제목과 내용을 이용해 중복을 안정적으로 합치도록 했습니다.
- 실제 API 호출 검사에서 공백뿐인 `OPENAI_API_KEY`를 유효한 키로 취급하지 않도록 했습니다.
- AI 검사 기록에 앱 버전과 소스 상태를 남기되 답변 원문은 저장하지 않도록 했습니다.
- GitHub 자동 검사에서 빈 데이터베이스에 전체 변경 이력을 적용하고, 나중에 적용하는 변경이 기존 데이터를 지우지 않는지 확인하도록 했습니다.

## 어떻게 확인했나요?

- 검색 결과 결합 집중 검사: `48/48` 통과
- 전체 계산·기능 자동 검사: `45`개 파일, `206`개 통과, 의도적으로 제외한 검사 `5`개
- 다섯 언어 집중 검색 질문: `30/30` 완료
- 넓은 질문은 첫 번째 검색 결과와 상위 8개 결과에서 필요한 문서를 모두 찾음
- 문서 이름을 지정한 질문은 상위 4개 결과에서 필요한 문서를 모두 찾음
- 여러 문서의 관계를 함께 봐야 하는 질문은 상위 8개 결과에서 필요한 경로를 모두 찾음
- 넓은 질문의 첫 검색 결과에 다른 종류 문서가 섞인 비율: `0.0`
- 실제 GPT-5.5 대화 경로: 정상 응답, 답변 완료, 발견된 문제 `0`
- 빈 PostgreSQL에서 전체 변경 이력 `29`개 적용 통과
- 나중에 적용하는 변경의 기존 데이터 보존 검사 통과

## 공개 범위와 보안 경계

공개 소스에는 이번 검색 수정, 자동 검사, 검사 도구, 버전 정보, 검토 보고서, 원문을 제외한 검사 요약을 포함했습니다.

API 키, `.env*`, 질문·답변 원문, 대화 내용, 비공개 평가 자료, 환자 정보, 비공개 원자료 연결 정보, 운영 DB 연결정보, 비공개 데이터베이스 변경 이력, Vercel 로컬 설정, 실행 기록과 캐시는 공개하지 않았습니다.

이번 패치는 AI가 코드를 자동 확정하거나 진단을 내리거나 임상가의 판단을 대신하게 만들지 않습니다. 본인의 API 키를 연결해 사용하는 BYOK(Bring Your Own Key) 방식과, 최종 판단을 사람이 맡는 HITL(Human-in-the-Loop) 원칙은 그대로 유지합니다.

## 남아 있는 한계

- 30개 집중 검색 질문은 검색 방식을 고치는 데 사용했으므로, 수정 과정과 완전히 분리해 처음 공개한 별도 시험 자료는 아닙니다.
- 넓은 해석 질문을 알아보는 다국어 규칙은 준비한 표현을 기준으로 하므로, 가능한 모든 자연어 표현을 빠짐없이 처리한다는 증명은 아닙니다.
- 실제 대화 경로 검사는 아직 하나의 가상 해석 사례만 다룹니다.
- 자동 검사는 정해 둔 프로그램 동작을 확인하며 임상적 정확성을 증명하지 않습니다.
- 실제 임상 유용성, 다국어 문장 품질, 안전성은 전문가의 사람 검토가 계속 필요합니다.
