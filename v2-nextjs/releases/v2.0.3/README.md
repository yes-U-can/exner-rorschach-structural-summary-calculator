# [2026-06-11] v2.0.3 버그 패치

v2.0.3은 v2.0.2 이후 확인된 참조 문서 페이지의 로딩 성능 문제와 BYOK 세션 처리의 사소한 결함을 수정한 버그 패치입니다. 화면 구성이나 사용 방식, AI 기능의 동작은 바꾸지 않았고, 같은 화면이 더 빠르고 안전하게 열리도록 코드 내부만 정리했습니다.

## 핵심 요약

- 참조 문서(`/ref`) 페이지가 다섯 언어 문서 전체(약 13MB)를 브라우저에 함께 내려보내던 문제를 수정했습니다. 참조 문서 화면에서 내려받는 JavaScript 총량이 약 1.4MB로 줄었습니다.
- 참조 문서 링크 주소를 전체 문서 데이터 없이 만들도록 분리하고, 5개 언어 1,015개 공개 문서의 링크가 이전과 동일한지 반복 검사했습니다.
- 손상된 BYOK 세션 쿠키가 전달된 경우 서버 오류(500) 대신 비활성 세션으로 안전하게 처리하도록 보강했습니다.
- BYOK 세션 상태 응답이 중간 캐시에 저장되지 않도록 `Cache-Control: no-store` 헤더를 추가했습니다.
- Next.js를 16.2.4에서 16.2.9로 업데이트해 프레임워크에 보고된 보안 권고(미들웨어 우회, CSP nonce 관련 XSS, 캐시 오염 등)를 해소했습니다. 업데이트 후 의존성 보안 점검(`npm audit`) 기준 알려진 취약점은 0건입니다.

## 영향받은 흐름

v2.0.2까지는 참조 문서 화면이 링크 주소를 만들기 위해 다섯 언어 문서 전체를 함께 불러왔습니다. 이 때문에 참조 문서 페이지를 여는 브라우저가 약 13MB의 JavaScript를 불필요하게 추가로 내려받았습니다.

계산기, 구조요약 결과, AI 대화 화면은 이 번들을 사용하지 않으므로 영향이 없었고, 참조 문서의 내용이나 링크가 잘못 표시된 것도 아닙니다. 다만 참조 문서 페이지의 첫 로딩이 회선 속도에 따라 불필요하게 길어질 수 있었습니다. v2.0.3에서는 이 부분을 성능 결함으로 보고 수정했습니다.

## 수정 내용

참조 문서 링크 주소를 문서 전체를 불러오지 않고 만들 수 있도록 분리했습니다. 분리 전후에 모든 공개 문서의 주소가 같은지 반복 확인했습니다. 이제 브라우저는 화면에 필요한 내용만 받고, 전체 참조 문서는 서버에서만 사용합니다.

BYOK 세션 쪽에서는 두 가지를 보강했습니다. 잘못된 퍼센트 인코딩이 섞인 세션 쿠키가 서버에 도착하면 이전에는 디코딩 단계에서 예외가 발생해 500 응답이 날 수 있었는데, 이제는 세션 없음으로 처리합니다. 또 세션 상태 조회 응답에 `Cache-Control: no-store`를 명시해 마스킹된 키 정보가 중간 캐시에 남지 않도록 했습니다.

마지막으로 Next.js 16.2.4에 보고된 보안 권고(미들웨어 우회, CSP nonce 관련 XSS, 캐시 오염, 서비스 거부 등)를 해소하기 위해 프레임워크를 16.2.9로 업데이트했습니다. 이 앱은 보안 헤더 전체를 미들웨어에서 적용하므로 미들웨어 우회 계열 권고가 특히 중요했습니다.
