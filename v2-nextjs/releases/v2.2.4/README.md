# [2026-07-18] v2.2.4 버그 패치

## 먼저 확인하실 점

v2.2.4는 **구조요약 계산식과 채점표의 입력 방식을 바꾸지 않은** 참조 문서·AI 검색·안전성 보완 패치입니다. 완성된 프로토콜의 구조요약 결과를 다시 계산할 필요가 없습니다. API 키 없이 계산기만 사용하는 경우에도 새로 입력해야 하는 피검자 정보는 없습니다.

이번 패치에서는 사람이 읽는 참조 문서와 AI가 검색하는 자료를 함께 다시 점검했습니다. 다섯 언어의 임상 용어는 단순 직역으로 맞추지 않고, 각 언어권의 전문 문헌과 학술 자료에서 실제로 쓰이는 표현을 우선했습니다. 문서의 제목과 배열도 알파벳순이 아니라 채점과 구조요약을 따라가기 쉬운 순서로 정리했습니다.

선택 기능인 코딩 도우미와 해석 도우미는 Exner Comprehensive System(CS) 범위 안에서만 답하도록 경계를 보강했습니다. 시스템 지침이나 내부 자료를 빼내려는 질문, 다른 검사 체계로 대화를 돌리려는 질문, 같은 요청을 짧은 시간에 과도하게 반복하는 경우도 함께 점검했습니다.

## 참조 문서를 어떻게 다듬었나요?

### 화면에서 읽기 쉬운 제목과 순서

참조 문서의 주소와 AI 내부 식별자는 그대로 두고, 화면에 보이는 제목만 각 언어의 임상 용어에 맞게 분리했습니다. 따라서 기존 링크는 계속 작동하면서도 사람이 보는 버튼에는 `scoring-input` 같은 내부 경로보다 `부호화`, `형태질(FQ)`, `특수지표`처럼 뜻을 알 수 있는 이름이 먼저 나타납니다.

상위 문서는 다음 흐름으로 배열했습니다.

1. 부호화
2. 해석
3. Upper Section
4. Lower Section
5. Special Indices

부호화 문서 안에서도 카드, 영역, 발달질(DQ), 결정인, 형태질(FQ), 쌍반응, 내용, 평범반응(P), 조직화 활동(Z), 점수, GHR/PHR, 특수점수의 실제 작업 흐름을 우선했습니다. 알파벳순은 같은 범주의 세부 항목을 찾을 때만 보조 기준으로 사용합니다.

### 다섯 언어의 문체와 용어

한국어, 영어, 일본어, 스페인어, 브라질 포르투갈어 문서는 각 언어에서 자연스러운 전문 문체로 통일했습니다. 종결 표현만 기계적으로 바꾼 것이 아니라 제목, 핵심 정의, 적용 조건, 주의점, 상호 참조가 같은 임상 개념을 가리키는지 확인했습니다.

대표적으로 다음 내용을 바로잡았습니다.

- 영어 PHR 문서에서 `ALOG`가 판정 순서의 초기 PHR 조건에 포함된다는 점을 명확히 했습니다.
- 영어와 스페인어의 자연 내용 부호에서 `Na`가 적용되면 같은 반응에 `Bt` 또는 `Ls`를 함께 부호화하지 않는 우선순위를 명확히 했습니다.
- 일본어 문서에서 `Ay`를 해부학적 내용이 아니라 문화·역사적 내용으로 설명하도록 바로잡았습니다.
- 한국어 문서에서 원시 빈도인 `S-`와 별도 비율인 `S-%`를 구분했습니다.
- 한국어 S-CON 문서에서 만 15세 이상 적용 경계와 12개 구성 기준을 다시 확인했습니다.
- 스페인어와 포르투갈어의 악센트, 성별·수 일치, 전문 문체를 정리했습니다.
- 작성 중 상태를 설명하던 문장, 내부 파일 경로, 잘못된 관련 문서 링크를 공개 본문과 AI 검색 자료에서 제거했습니다.

이 변경은 임상가가 원반응과 Inquiry를 보고 수행하는 부호화를 대신하지 않습니다. 참조 문서는 부호의 정의와 구분 기준을 확인하는 보조 자료이며, 개별 반응의 최종 부호화는 계속 인간 채점자의 책임입니다.

## AI가 참조 문서를 찾는 과정도 함께 고쳤습니다

참조 문서 1,015개를 현재 본문으로 다시 만들고, AI가 검색할 수 있는 5,589개의 작은 검색 단위로 나누어 OpenAI `text-embedding-3-large` 벡터를 모두 갱신했습니다.

각 검색 단위에는 본문 내용의 SHA-256 해시를 함께 저장합니다. 문서 주소는 같지만 내용만 바뀐 경우에도 예전 벡터를 최신 자료로 잘못 판단하지 않도록 하기 위한 값입니다. 최종 확인에서 누락된 벡터, 오래된 벡터, 본문 해시가 맞지 않는 벡터는 모두 0개였습니다.

검색 경로에서는 두 가지 실제 문제를 추가로 고쳤습니다.

- Cn과 WSumC의 관계를 묻는 짧은 질문에서도 Cn이 들어가는 화면값과 들어가지 않는 WSumC 설명을 함께 찾도록 했습니다.
- `Na`, `Bt`, `Ls`의 우선순위를 묻는 질문이 세 내용 부호의 일반 설명만 찾고 정확한 우선순위 문장을 놓치지 않도록 했습니다.

검색 정확성 검사는 정답 문서가 첫 결과와 상위 결과 안에 들어오는지, 이름을 직접 말한 문서를 찾는지, 서로 관련된 문서를 함께 찾는지, 부호화 질문에 해석 문서가 불필요하게 섞이지 않는지를 나누어 확인했습니다.

## 코딩 도우미와 해석 도우미의 범위

두 도우미는 다음 원칙을 따릅니다.

- Exner Comprehensive System(CS)의 부호화와 구조요약 질문에만 답합니다.
- R-PAS, MMPI처럼 별도의 검사 체계나 일반 상담·진단 질문으로 답변 범위를 넓히지 않습니다.
- 시스템 지침, 숨겨진 설정, 내부 평가 기준, 원문 검색 자료, API 키, 쿠키, 세션 정보의 공개·재구성 요청을 거절합니다.
- 해석에 연령이 실제로 필요한 경우에는 AI 대화 안에서 필요한 이유를 설명하고 질문할 수 있지만, 계산기 본체에 연령 입력을 요구하지 않습니다.
- 구조요약만으로 진단이나 위험을 확정하지 않으며, 면담·행동관찰·원자료와 임상가의 판단을 우선합니다.

프롬프트 인젝션은 겉으로는 로샤 질문처럼 보이면서 앞선 지침을 무시하라고 요구하는 방식까지 포함해 검사했습니다. 거절 답변에는 단순히 “답할 수 없다”는 말만 남기지 않고, 답변 가능한 정확한 범위인 Exner CS 부호화 또는 구조요약 질문으로 돌아오도록 안내하게 했습니다.

## AI 요청의 과도한 반복 방지

활성화된 암호화 BYOK 세션마다 `/api/chat` 요청을 1분에 12회, 1시간에 120회로 제한했습니다. 제한을 넘으면 OpenAI 호출을 시작하지 않고 기다려야 하는 시간을 포함한 `429` 응답을 보냅니다.

이 횟수 확인은 서버 인스턴스의 짧은 메모리 안에서만 처리합니다. API 키, 질문, 답변, 구조요약 원문, IP 주소, 임상 내용은 이 제한을 위해 데이터베이스에 저장하지 않습니다. 서버 인스턴스가 바뀌면 메모리 기록도 이어지지 않으므로, 이 장치는 비용 폭주와 실수성 반복을 줄이는 앱 내부의 1차 방어입니다. 더 넓은 공격은 Vercel의 외곽 방화벽과 OpenAI 계정의 사용 한도를 함께 사용해야 합니다.

좋아요·싫어요 평가의 별도 데이터베이스 제한과 180일 보관 정책은 v2.2.3 기준을 그대로 유지합니다.

## 화면과 서비스 설명에서 바뀐 점

- 왼쪽 사이드바는 뒤의 본문이 비치지 않는 불투명 배경으로 고정했습니다.
- 사이드바가 접힌 상태에서 언어 메뉴를 열 때 메뉴가 잘리거나 본문 위에 어긋나던 문제를 고쳤습니다.
- 참조 문서 버튼에는 다섯 언어의 사람이 읽는 제목과 같은 작업 순서를 적용했습니다.
- 채점 화면에 들어올 때마다 새 데이터, 샘플 데이터, 저장 데이터 중 시작 방식을 다시 선택하는 창이 열리도록 복원했습니다.
- 참조 문서의 코드형 핵심 구절은 라이트·다크 모드 모두에서 구분하기 쉬운 붉은색으로 표시합니다.
- 코딩 도우미에서 이전 대화를 읽으려고 위로 이동하면 나타나는 아래쪽 화살표를 입력 영역 바로 위에 배치했습니다. 긴 답변에서도 화살표가 대화창 한가운데를 가리지 않습니다.
- 버전 기록의 Next.js와 Google Apps Script 목록은 처음 들어왔을 때 접힌 상태로 표시합니다.
- 서비스 명칭은 `Exner 로샤 종합체계 구조요약 계산기`로 통일했습니다.
- MOW(모오)는 기획, 제작, 배포, 운영, 유지보수를 맡습니다. 서울임상심리연구소(Seoul Institute of Clinical Psychology, SICP)는 초기 계산 로직 이관 검토, 계산 결과 대조, 실제 사용 관점의 임상 검수를 맡습니다.

채점표의 열, 드롭다운, 계산 버튼, 확대·축소 조작과 구조요약 결과 화면은 이번 패치에서 바꾸지 않았습니다. 모바일 전용 UI/UX 최적화도 이번 범위에 포함하지 않았습니다.

공개 아카이브의 v2.1.8-v2.2.4 표시 날짜는 당시 작업일에 맞춰 각각 하루 앞당겼습니다. 이 정리는 패치노트와 아카이브에 보이는 릴리즈 날짜만 맞춘 것이며, 소스 커밋 시각과 검사 보고서 작성일은 바꾸지 않았습니다.

## 다섯 언어 용어 검토에 사용한 공개 기준

단일 출처를 모든 언어의 정답으로 삼지 않았습니다. 각 언어의 전문 용례를 먼저 확인하고, Comprehensive System의 부호와 식별자는 그대로 유지했습니다.

- 한국어: [KCI - Exner의 종합체계에 근거한 한국형 아동 로르샤하 종합체계의 구성](https://www.kci.go.kr/kciportal/landing/article.kci?arti_id=ART001392063), [KCI - 로샤 검사에 나타난 북한이탈주민의 대처와 방어](https://www.kci.go.kr/kciportal/landing/article.kci?arti_id=ART001391524)
- 영어: [International Rorschach Institute manuals](https://www.rorschach-institute.org/manuals.html), [Meyer and Viglione, An Introduction to Rorschach Assessment](https://www.utoledo.edu/al/psychology/pdfs/meyer/MeyerViglione2008IntroRorschach.pdf)
- 일본어: [誠信書房 - 包括システムによるロールシャッハ臨床](https://www.seishinshobo.co.jp/book/b88274.html)
- 스페인어: [Sociedad Española de Rorschach y Métodos Proyectivos](https://www.rorschach.es/index.php/programas-de-los-cursos), [CHESSSS](https://rorschachspain.org/chessss/), [Manual de codificación del Rorschach para el Sistema Comprehensivo](https://www.psimatica.com/tienda/psicodiagnostico/23-manual-de-codificacion-del-rorschach-autor-john-exner.html)
- 브라질 포르투갈어: [SciELO - Localização e qualidade formal do Rorschach-SC no Brasil](https://www.scielo.br/j/pusf/a/kFHxFGKH3qx9gdVtyC6nqWS/), [SciELO - Indícios de validade do déficit relacional no Método de Rorschach](https://www.scielo.br/j/pusf/a/6Xy8zSJGCNq49BWjXRpYNhx/)
