# [2026-07-03] v2.1.5 버그 패치

v2.1.5는 AI 답변 방식이 바뀔 때마다 같은 품질·개인정보·보안 검사를 빠뜨리지 않고 실행하도록 만든 패치입니다.

UI/UX는 변경하지 않았습니다. 개발 컴퓨터와 GitHub의 자동 검사에서 같은 네 단계를 순서대로 실행하고, 하나라도 실패하면 릴리즈 준비를 중단하도록 했습니다.

## 핵심 요약

- AI 답변 기준, 저장된 검사 기록, 사람 검토 기록, 비밀정보 노출 여부를 한 번에 확인하는 종합 검사 명령을 추가했습니다.
- 실제 실행 전에 검사 순서만 미리 볼 수 있고, 결과를 JSON 또는 Markdown 문서로 남길 수 있는 종합 검사 도구를 추가했습니다.
- 종합 검사 도구 자체를 확인하는 자동 검사를 추가했습니다.
- 종합 검사 도구가 답변 기준 검사에서 빠지지 않도록 연결했습니다.
- GitHub의 보안 검사도 외부 프로그램 취약점 검사 전에 이 AI 종합 검사를 실행하도록 했습니다.
- 검사 내용과 결과는 `docs/ai-evals/2026-07-03-v2.1.5-ai-release-gate-report.md`에 기록했습니다.
- footer, package, version archive 메타데이터를 v2.1.5로 갱신했습니다.

## 릴리즈 전에 확인하는 네 단계

새 종합 검사는 다음 네 항목을 실행합니다.

1. 코딩·해석 도우미가 정해 둔 답변 범위와 임상가 최종 판단 원칙을 지키는지 확인합니다.
2. 저장된 검사 기록에 답변 중단이나 개인정보 노출이 없는지 확인합니다.
3. 사람이 검토한 답변 기록이 정해 둔 기준을 통과했는지 확인합니다.
4. 공개 소스에 API 키나 데이터베이스 연결정보 같은 비밀정보가 섞이지 않았는지 확인합니다.

필수 단계가 실패하면 즉시 중단되므로 개발 컴퓨터와 GitHub 자동 검사에서 같은 기준을 사용할 수 있습니다.

## 어떻게 확인했나요?

AI 종합 검사, 코드 형식, 배포용 페이지 생성, 보안 검사를 통과했습니다.

<details>
<summary><strong>개발자가 같은 검사를 다시 실행할 때 사용하는 명령</strong></summary>

- `npm run ai:evaluate-contracts`
- `npm run ai:evaluate-artifacts`
- `npm run ai:evaluate-human-records -- --require-pass`
- `npm run security:secrets`
- `npm run ai:release-gate`
- `npm run lint`
- `npm run build`
- `npm run security:check`
- `git diff --check`

</details>

당시 보안 검사는 심각도가 높은 취약점을 차단하는 기준으로 통과했습니다. 다만 `gray-matter`가 간접적으로 사용하는 `js-yaml`에서 중간 수준의 보안 권고가 남아 있었고, 이 항목은 v2.1.6에서 해결했습니다.

## 공개 범위

공개 소스에는 종합 검사 도구, 자동 검사, 설명 문서와 릴리즈 정보를 포함했습니다.

공개 아카이브에는 API 키, `.env*` 파일, Vercel 로컬 설정, 답변·질문·대화 원문, 비공개 운영 노트를 포함하지 않았습니다.
