# [2026-05-21] v2.0.2 버그 패치

v2.0.2는 v2.0.1 이후 확인된 구조요약 CSV 관련 문제를 수정한 버그 패치입니다. 큰 제품 방향이나 AI 기능의 사용 방식은 바꾸지 않았고, 결과 화면에서 복사하거나 내려받는 데이터가 화면의 구조요약 계산 결과를 더 정확하게 반영하도록 정리했습니다.

## 핵심 요약

- 데이터 다운로드 모달의 한국어 항목명 `입력값 원자료 CSV`를 `점수계열 원자료 CSV`로 수정했습니다.
- `구조요약 값 복사하기` 버튼으로 복사되는 CSV 문자열에서 중복 헤더와 누락 항목이 생길 수 있던 문제를 수정했습니다.
- `구조요약 수치 CSV` 다운로드도 같은 안정화된 CSV 생성 로직을 사용하도록 정리했습니다.
- 해석 AI 대화에 붙여넣는 구조요약 CSV가 제품이 기대하는 형식으로 검증되도록 테스트를 추가했습니다.
- API 키로 로그인한 AI 세션에서 AI 제공자 키 오류가 발생했을 때 세션 시작 팝업이 반복해서 뜨던 문제를 수정했습니다.

## 영향받은 흐름

v2.0.1의 해석 도우미는 결과 화면에서 `구조요약 값 복사하기` 버튼으로 복사한 값을 해석 AI 화면의 구조요약 수치 입력란에 붙여넣고 대화를 시작하는 방식입니다.

기존 구현에서는 이 복사 문자열을 만들 때 구조요약 결과 객체를 일반적인 순회 방식으로 훑었습니다. 그 결과 `D`, `Zf`, `Zd`, `GHR`, `PHR`처럼 여러 구역에서 쓰이는 이름이 CSV 헤더에서 중복될 수 있었고, `Single`, `Contents`, `Form Quality`, `Special Scores`, `Approach`, `Blends`처럼 화면에 표시되는 일부 구조요약 항목이 빠질 수 있었습니다.

따라서 API 키 로그인이나 AI 대화 기능 자체가 고장난 것은 아니지만, 해석 AI에 전달되는 구조요약 입력 자료가 불완전하거나 모호할 수 있었습니다. v2.0.2에서는 이 부분을 버그로 보고 수정했습니다.

## 수정 내용

구조요약 CSV 생성 방식을 명시적 열 목록 기반으로 바꿨습니다. 이제 복사와 다운로드는 같은 열 정의를 사용하며, 위치 구역의 `D`와 Lower Section의 `D`처럼 이름이 겹칠 수 있는 값은 `Loc_D`와 `D`처럼 구분합니다.

점수계열 원자료 CSV는 계산에 실제 참여하는 채점 행만 내보내도록 정리했습니다. 카드가 비어 있는 임시 행이나 미완성 행이 CSV에 섞여 사용자를 혼란스럽게 만들 가능성을 줄였습니다.

해석 도우미 서버 검증도 새 구조요약 CSV 열 목록을 기준으로 맞췄습니다. 사용자가 결과 화면에서 복사한 정상 구조요약 CSV를 붙여넣으면 통과하고, 일반 문장이나 잘못된 프롬프트 형태의 텍스트는 계속 거부합니다.
