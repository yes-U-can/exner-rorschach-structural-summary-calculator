# [2026-04-27] v2.0.1 버그 패치

v2.0.1은 v2.0.0 공개 이후 실제 사용 점검 과정에서 발견한 문제를 정리한 안정화 패치입니다. 큰 제품 방향은 바꾸지 않았습니다. 이 버전에서도 웹앱의 핵심은 구조요약 계산, 참조 문서, BYOK 기반 코딩 도우미와 해석 도우미입니다.

이번 패치의 목표는 기능을 더 늘리는 것이 아니라, v2.0.0에서 남아 있던 불필요한 흐름을 덜어내고 사용자가 실제로 대화하고 검토하는 흐름을 더 안정적으로 만드는 것이었습니다. 특히 AI가 웹앱 안에서 임상가의 판단을 대신하는 버튼이 아니라, 사용자가 묻고 검토하는 대화형 보조 도구로만 남도록 정리했습니다.

## 핵심 요약

- 해석 도우미의 입력 흐름을 파일 첨부 중심에서 구조요약 수치 입력 중심으로 정리했습니다.
- 코딩 도우미와 해석 도우미에서 AI가 너무 길게 말하거나, 사용자가 제어하기 어려운 방식으로 화면을 움직이던 문제를 줄였습니다.
- AI 자동 채우기 흐름을 제거했습니다. 부호화와 해석은 사용자가 AI와 대화하며 검토하는 방식만 남겼습니다.
- BYOK 세션, API 키 제공자 검증, 모달 겹침, API 키 없음 안내 흐름을 정리했습니다.
- 다크모드에서 토스트와 시스템 안내 말풍선이 투명하게 보이거나 읽기 어려웠던 문제를 수정했습니다.
- 참조 문서의 `+`, `-`, `v/+` 같은 부호 경로가 404로 이어지던 문제를 안전한 URL slug로 정리했습니다.
- 서비스 소개, 이용약관, 개인정보처리방침, 버전 아카이브 문서를 현재 기능 기준으로 다시 맞췄습니다.
- 공개 전시용 소스 스냅샷을 최신 private 소스 기준으로 갱신하되, 운영 환경변수와 비공개 작업 노트는 포함하지 않았습니다.

## 해석 도우미 입력 흐름 정리

v2.0.0에서는 해석 도우미가 구조요약 CSV 파일을 첨부하는 흐름으로 설명되어 있었습니다. 실제 테스트 과정에서 파일 첨부 방식은 사용자가 어떤 파일을 넣어야 하는지 헷갈릴 수 있고, AI가 파일 내용을 안정적으로 같은 방식으로 읽지 못할 여지가 있었습니다.

v2.0.1에서는 해석 도우미가 결과 화면에서 복사한 구조요약 수치를 별도 입력란에 붙여넣는 흐름으로 정리되었습니다. 사용자는 결과 화면의 구조요약 값을 복사한 뒤, 해석 도우미 입력창 왼쪽의 작은 구조요약 수치 입력란에 붙여넣고 질문을 시작합니다.

구조요약 수치 입력란은 원문 전체를 그대로 화면에 펼쳐 보이지 않습니다. 값이 정상적으로 들어가면 `입력됨`으로 표시하여 화면이 지저분해지지 않게 했습니다. 사용자가 직접 지우거나 새 값으로 바꾸기 전까지는 같은 브라우저 세션 안에서 유지되도록 했습니다.

또한 해석 도우미가 아무 자료나 받지 않도록, 결과 화면에서 복사한 구조요약 값 형식에 맞는 입력만 사용하도록 제한했습니다. 사용자가 엉뚱한 텍스트를 넣고 대화를 시작하려 하면, 결과 화면의 구조요약 값 복사 기능을 사용하라는 안내를 띄우도록 했습니다.

## 해석 도우미 응답 품질 조정

해석 도우미의 답변은 사용자가 이어서 대화할 수 있는 길이와 형식으로 조정했습니다. v2.0.0 직후에는 답변이 지나치게 길어지거나, 중간에서 끊기거나, 수치를 한 줄에 너무 많이 나열해 읽기 어려운 경우가 있었습니다.

v2.0.1에서는 해석 도우미가 한 번의 답변에서 너무 많은 주제를 한꺼번에 펼치지 않도록 지침을 정리했습니다. 수치를 제시할 때에는 `근거 수치`와 `해석 가설`을 나누어 쓰고, 한 줄에 너무 많은 변인을 몰아넣지 않도록 했습니다.

또한 사용자가 로샤 검사를 다루는 임상심리사 또는 수련생이라는 점을 고려해, 불필요하게 IT 용어처럼 들릴 수 있는 표현을 줄였습니다. 예를 들어 `프로토콜`이라는 표현은 가능한 한 `검사자료`, `응답자료`, `구조요약 자료`처럼 더 직관적인 표현으로 바꾸도록 했습니다.

## 코딩 도우미 흐름 정리

코딩 도우미는 채점 화면에서 사용자가 반응을 부호화할 때 대화로 도움을 받는 기능입니다. v2.0.1에서는 코딩 도우미가 자동으로 행을 채우거나 버튼으로 필드를 적용하는 흐름을 제거했습니다.

이제 코딩 도우미는 사용자가 선택한 행과 전체 시트 맥락을 바탕으로 후보와 근거를 대화로 설명하는 역할만 합니다. 사용자는 AI의 설명을 참고하되, 실제 부호화 값은 직접 검토하고 입력해야 합니다.

채점 화면에서 `Ctrl/Cmd+J`로 코딩 도우미를 열 수 있는 흐름은 유지했습니다. 아무 행도 선택하지 않은 상태에서는 특정 중심 행 없이 전체 시트만 맥락으로 사용하고, 선택된 행이 있으면 그 행을 더 중요한 맥락으로 다룹니다.

## AI 자동 채우기 제거

v2.0.0 이후 점검 중 `AI 자동 채우기` 기능은 제거하기로 결정했습니다. 이 기능은 반응 메모와 카드 정보를 바탕으로 현재 행의 부호화 필드를 한 번에 채우도록 시도하는 흐름이었습니다.

하지만 로샤 부호화는 단순 자동완성보다 임상가의 검토와 판단이 중요한 작업입니다. 버튼 한 번으로 값을 채우는 기능은 편리해 보일 수 있지만, 사용자가 AI의 제안을 충분히 검토하지 않고 받아들이게 만들 위험이 있었습니다. 따라서 HITL, 즉 사람이 판단 과정 안에 남아 있어야 한다는 원칙에 맞게 자동 채우기 흐름을 제거했습니다.

그 결과 v2.0.1의 AI 기능은 더 단순해졌습니다. 웹앱 안의 AI는 코딩 도우미와 해석 도우미, 이 두 가지 대화형 보조 기능만 제공합니다.

## BYOK 세션과 API 키 흐름 안정화

BYOK API 키는 서버 DB에 저장하지 않고 이번 로그인 세션에서만 사용한다는 원칙을 유지했습니다. v2.0.1에서는 이 흐름을 더 명확하게 만들었습니다.

OpenAI 탭에 Google API 키를 넣거나, Google 탭에 OpenAI API 키를 넣는 실수를 막기 위해 기본적인 제공자 형식 검증을 추가했습니다. 이 검증은 사용자의 키를 별도로 기억하거나 도용 여부를 추적하기 위한 것이 아니라, 잘못된 탭에 키를 입력하는 단순 실수를 줄이기 위한 최소한의 검사입니다.

또한 API 세션 종료 버튼, API 키 입력 모달, 해석 도우미 진입 가드가 서로 겹치면서 화면이 이상하게 보이던 문제를 수정했습니다. API 키가 없으면 해석 도우미 화면으로 우회 진입하지 못하도록 했고, 필요한 경우 계정 관리 또는 API 키 입력 모달로 안내하도록 정리했습니다.

## 모델 선택 원칙 정리

사용자는 모델을 직접 고르지 않습니다. 웹앱은 출시된 지 최소 한 달 이상 지나 안정성이 확인된 모델 중에서 서비스가 정한 최신 모델을 자동으로 사용합니다.

v2.0.1 기준 공개 문서와 UI에서는 이 원칙을 중복 안내하지 않도록 줄였습니다. 모델 선택 안내는 필요한 곳에만 남기고, 사용자가 모델을 고를 수 있는 것처럼 오해할 수 있는 문구는 제거했습니다.

## 다크모드와 시스템 안내 수정

다크모드에서 시스템 안내 말풍선이나 우측 상단 토스트가 투명하게 보이거나, 배경과 섞여 잘 읽히지 않는 문제가 있었습니다. v2.0.1에서는 토스트와 채팅 안내 메시지의 배경, 테두리, 글자색을 더 명확한 표면 색으로 정리했습니다.

이 수정은 단순히 색을 어둡게 바꾸는 것이 아니라, 라이트모드와 다크모드에서 같은 정보 위계를 유지하도록 표면 토큰을 다시 맞춘 작업입니다. 특히 성공 안내, 경고 안내, 시스템 안내가 화면 요소 뒤로 숨거나 배경과 섞이지 않도록 조정했습니다.

## 채팅 UI 정리

코딩 도우미와 해석 도우미 모두에서 불필요한 안내 문구와 중복 메시지를 줄였습니다. AI 모델명, 참조 문서 목록, 필드 적용 버튼처럼 사용자의 대화 흐름을 방해하거나 과한 기능처럼 보이는 요소를 제거했습니다.

AI가 답변하는 동안 화면이 강제로 계속 아래로 내려가 사용자가 스크롤을 제어하기 어려웠던 문제도 줄였습니다. 이제 사용자가 이전 내용을 보고 있는 상황에서는 자동 스크롤이 과하게 개입하지 않도록 정리했습니다.

로딩 중 말풍선의 점 애니메이션도 눈에 덜 거슬리도록 완화했습니다. 대화형 도구는 사용자가 오래 바라보는 화면이므로, 작은 움직임이라도 피로감을 줄 수 있다는 점을 반영했습니다.

## 참조 문서 라우팅 수정

참조 문서에는 `+`, `-`, `v/+`처럼 부호 자체가 문서 이름에 포함되는 항목이 있습니다. v2.0.0에서는 이 값들이 URL 경로로 그대로 들어가면서, 일부 링크가 잘못 쪼개지거나 404로 이어지는 문제가 있었습니다.

v2.0.1에서는 내부 문서 식별자와 브라우저 URL용 slug를 분리했습니다. 내부적으로는 원래 의미를 유지하되, 브라우저 주소에서는 안전한 slug를 사용하도록 바꿨습니다. 이로써 `[부호화/발달질] v`, `[부호화/발달질] v/+`, `[부호화/형태질] +`, `[부호화/형태질] -` 같은 항목으로 이동할 때 잘못된 문서로 가거나 404가 뜨는 문제를 줄였습니다.

## 문서와 정책 문구 정리

서비스 소개, 이용약관, 개인정보처리방침은 현재 남아 있는 기능 기준으로 다시 맞췄습니다. 결제, 크레딧, 스토어, 플랫폼 제공 AI처럼 실제 공개 범위에서 제거된 기능을 암시하는 문구는 정리했습니다.

서비스 소개에서는 이 웹앱이 어떤 목적과 원칙으로 설계되었는지, BYOK를 왜 선택했는지, 참조 문서를 왜 공개하는지, 그리고 Human-in-the-Loop 원칙과 어떤 관련이 있는지를 더 자연스럽게 설명하도록 정리했습니다.

개인정보처리방침과 이용약관은 제품 소개문이 아니라 각각의 역할에 맞게 정리했습니다. 개인정보처리방침은 데이터 처리와 보관, 삭제 기준을 중심으로 두고, 이용약관은 이용 조건과 책임 범위에 집중하도록 했습니다.

## 버전 아카이브와 공개 저장소 정리

v2.0.1 패치노트를 공개 전시용 저장소에 추가했습니다. v1 GAS 릴리즈 기록과 v2 Next.js 릴리즈 기록을 한 저장소 안에서 함께 볼 수 있도록 유지했습니다.

공개 저장소의 `v2-nextjs/source`는 최신 private 소스를 기준으로 갱신했습니다. 다만 운영 환경변수, Vercel 설정, 로컬 로그, 캐시, `node_modules`, 비공개 작업 노트, 과거 실험 문서는 포함하지 않았습니다.

공개 목적은 사용자가 웹앱의 작동 방식과 설계 의도를 확인할 수 있게 하는 것입니다. 운영 비밀이나 불필요한 개발 흔적을 전시하는 것이 목적은 아닙니다.

## 공개 범위

이번 공개 스냅샷에는 다음 자료를 포함했습니다.

- Next.js 앱 소스코드
- UI 컴포넌트
- 로샤 구조요약 계산 로직
- BYOK AI 도우미 관련 코드
- 참조 문서 코퍼스 생성 결과
- 참조 문서 초안 중 공개용 `reference-authoring/drafts`
- 다국어 locale 파일
- 테스트 코드
- 빌드 설정과 실행에 필요한 설정 예시

다음 자료는 공개 스냅샷에 포함하지 않았습니다.

- 실제 운영 환경변수
- API 키
- Vercel 프로젝트 설정
- 로컬 로그
- 캐시와 빌드 산출물
- `node_modules`
- 과거 Prisma migration 이력
- 비공개 작업 노트
- 개발 중간 인계서와 운영 메모

## 검증

v2.0.1 공개 전 private 소스에서는 참조 문서 생성, 런타임 준비 검사, i18n 검사, TypeScript 검사, lint, 테스트, 빌드, secret scan을 기준으로 점검했습니다.

공개 전시용 저장소로 옮길 때에는 운영 환경변수와 로컬 전용 파일이 포함되지 않도록 whitelist 방식으로 동기화했습니다. 즉, 공개에 필요한 폴더와 파일만 골라 복사하고, `.env.local`, `.env.production.local`, `.next`, `.vercel`, `node_modules`, 로그 파일, 비공개 작업 노트는 복사 대상에서 제외했습니다.
