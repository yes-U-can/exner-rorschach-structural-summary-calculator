# [2026-06-29] v2.1.3 버그 패치

v2.1.3은 AI 화면을 바꾸지 않고, v2.1.2에서 저장한 실제 API 호출 결과가 빠짐없이 기록됐는지 다시 확인할 수 있게 만든 품질 강화 패치입니다.

UI/UX는 유지하고, 자동 검사 결과와 사람이 답변 유용성을 평가하는 기준표, 공개 문서, 버전 기록을 정리했습니다.

## 핵심 요약

- 저장된 실제 API 호출 기록을 다시 검사하는 `ai:evaluate-artifacts` 명령을 추가했습니다.
- 기록 파일을 읽을 수 있는지, 개별 결과와 전체 합계가 맞는지, API 키처럼 보이는 값이나 질문·답변 원문이 들어 있지 않은지 확인하도록 했습니다.
- 최종 통과 기록은 발견된 문제와 실행 실패가 모두 0이어야 인정하도록 기준을 정했습니다.
- 이 검사 도구 자체가 다시 깨지지 않는지 확인하는 반복 검사를 추가했습니다.
- 자동 검사만으로 판단하기 어려운 답변의 유용성을 사람이 같은 기준으로 평가할 수 있도록 사람 검토 기준표를 추가했습니다.
- AI 답변 방식이 바뀔 때 반드시 거쳐야 할 검사를 설계 원칙과 릴리즈 점검표에 명시했습니다.
- 공개 README에서 AI 품질 검사, 사람 검토 기준표, v2.1.2 사례를 바로 찾을 수 있도록 링크를 보강했습니다.
- 푸터 버전, i18n 버전 문자열, package version, 버전 아카이브를 v2.1.3으로 갱신했습니다.

## AI 답변을 확인하는 세 단계

이번 패치 이후 AI 관련 변경은 다음 세 층으로 검토할 수 있습니다.

1. **답변 기준 확인:** 필요한 내용과 임상적 한계를 지켰는지 자동으로 확인합니다.
2. **저장 기록 확인:** 검사 결과가 온전하고 개인정보나 원문이 섞이지 않았는지 확인합니다.
3. **사람 검토:** 자동 검사로 판단하기 어려운 실제 유용성을 임상가 관점의 기준표로 평가합니다.

이 구조는 "AI가 좋아졌다"는 주장보다, 어떤 기준을 통과했는지 남기는 쪽에 초점을 둡니다.

## 어떻게 확인했나요?

답변 기준, 저장 기록, 다국어 문구, 비밀정보 검사를 통과한 뒤 공개 소스를 갱신했습니다.

<details>
<summary><strong>개발자가 같은 검사를 다시 실행할 때 사용하는 명령</strong></summary>

- `npm run ai:evaluate-contracts`
- `npm run ai:evaluate-artifacts`
- `npm run i18n:audit`
- `npm run security:secrets`
- `git diff --check`

</details>

추가로 v2.1.x의 누적 변경 뒤에도 계산기, 참조 문서, AI 도우미의 기존 기능이 그대로 작동하는지 전체 검사를 다시 실행했습니다. 실행 가능한 108개 검사는 모두 통과했고, 외부 환경이 필요한 3개 검사는 별도로 제외했습니다.

## 공개 범위

공개 소스에는 AI 답변 검사 도구, 자동 검사, 저장 기록 확인 문서, 사람 검토 기준표, 릴리즈 점검표, 설계 원칙 문서를 포함했습니다.

API 키, `.env*`, Vercel 로컬 설정, 답변 원문, 질문 원문, 비공개 작업 노트는 포함하지 않았습니다.
