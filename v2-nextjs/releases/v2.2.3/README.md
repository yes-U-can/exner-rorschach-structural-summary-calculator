# [2026-07-17] v2.2.3 버그 패치

## 먼저 확인하실 점

v2.2.3은 **구조요약 계산식과 화면 배치를 바꾸지 않은** 문서·검색 노출·피드백 보안 패치입니다. 기존 구조요약 결과를 다시 계산할 필요가 없고, 계산기 입력 항목이나 AI 사용 여부도 달라지지 않습니다.

이번 패치에서는 공개 문서를 임상심리사가 먼저 이해할 수 있는 순서로 다시 정리했습니다. 검색결과와 링크 미리보기에는 다섯 언어별 제목·설명을 제공하고, 선택형 AI 응답 평가가 짧은 시간에 과도하게 반복돼 데이터베이스에 부담을 주지 않도록 방어 장치를 추가했습니다.

## 검색결과와 링크 미리보기

홈 화면의 검색·공유용 제목은 다섯 언어 모두 `Yes, U Can!`로 통일했습니다. 한국어 설명은 다음 문장으로 고정했습니다.

> 회원가입, 설치, 결제가 필요 없는 Exner Rorschach 종합체계(Comprehensive System) 검사 구조요약 계산기입니다. 오픈소스이며, 본 서비스는 전문가의 임상 판단을 대체하지 않습니다.

영어·일본어·스페인어·포르투갈어에서도 같은 의미가 전달되도록 각각 번역했습니다. 각 언어 주소에는 자기 자신을 가리키는 대표 주소와 다른 네 언어 주소를 함께 표시하고, 사이트맵에도 다섯 언어 페이지를 모두 넣었습니다.

공식 대표 주소는 `https://exner.yesucan.co.kr/`로 고정했습니다. Vercel의 기본 주소나 미리보기 주소에서 앱이 실행되더라도 검색엔진과 링크 미리보기에는 공식 도메인이 대표 주소로 전달됩니다.

일반 검색 설명과 링크 미리보기가 같은 제목·설명을 사용하도록 HTML 메타데이터, Open Graph, Twitter Card, WebSite 구조화 데이터를 함께 맞췄습니다. 이 변경은 페이지에 보이는 문구나 화면 배치를 바꾸지 않습니다.

검색엔진과 메신저는 이전 정보를 일정 기간 캐시할 수 있고, Google은 검색어에 따라 제목이나 설명을 일부 다시 구성할 수 있습니다. 따라서 배포 직후 모든 화면이 동시에 바뀐다고 보장할 수는 없지만, 웹앱이 제공하는 공식 후보 정보는 위 내용으로 일치시켰습니다.

## AI 응답 평가 데이터베이스 보호

좋아요·싫어요 평가는 계속 선택 기능입니다. 질문, 답변, 반응 메모, 구조요약 원문, API 키, 계정 식별자, IP 주소, 브라우저 정보는 앱의 피드백 데이터베이스에 저장하지 않습니다.

과도한 반복 요청을 막기 위해 다음 장치를 추가했습니다.

- 한 요청은 2 KiB를 넘으면 본문을 끝까지 읽기 전에 거부합니다.
- 암호화된 24시간 AI 세션마다 서버 안에서만 만드는 일방향 식별값을 사용합니다. 이 값으로 API 키나 대화 내용을 복원할 수 없습니다.
- 한 세션에서는 1분에 최대 10회, 24시간 동안 최대 60회까지만 평가를 기록합니다.
- 제한을 넘으면 잠시 기다린 뒤 다시 시도하도록 `429` 응답과 대기 시간을 돌려줍니다.
- 데이터베이스 연결 수와 대기 시간을 제한하고, 만료된 평가·제한 기록은 작은 묶음으로 정리합니다.
- 세션을 반복해서 새로 만드는 우회까지 줄이기 위해 `/api/chat/feedback`와 `/api/byok/session`을 합쳐 IP당 60초에 30회로 제한하는 Vercel Firewall 규칙을 운영에 게시했습니다.

피드백 자체는 기존과 같이 최대 180일 보관합니다. 새로 추가한 일방향 세션 식별값과 횟수 기록은 AI 세션의 최대 수명인 24시간까지만 유지합니다.

## 공개 문서와 릴리즈 날짜

v1 GAS 패치노트 전체에서 사용한 설명 방식을 기준으로, v2 문서도 다음 순서로 읽히도록 정리했습니다.

1. 임상가가 먼저 알아야 할 영향과 재계산 필요 여부
2. 문제가 나타나는 조건과 구체적인 예
3. 무엇을 고쳤고 어떤 한계가 남았는지
4. 개발자가 재현할 수 있는 수치·명령·파일 위치

과거 패치노트의 설명을 바로잡아야 하는 경우에는 기존 기록을 지우지 않고, 날짜가 표시된 정정 안내나 후속 기록을 덧붙여 당시 내용과 현재 기준을 함께 확인할 수 있도록 했습니다.

새벽 작업이 자정을 오간 버전은 커밋 시각이 아니라 당시 운영상 선언한 **Asia/Seoul 릴리즈 날짜**를 기준으로 아카이브 표기를 통일했습니다. 소스 커밋, 검사 보고서 작성일, 실제 계산 변경 내용은 바꾸지 않았습니다.

## 버전 이름을 사용하는 기준

- `v1.0.0 → v2.0.0`처럼 첫 숫자가 바뀌는 경우는 메이저 릴리즈입니다.
- `v2.1.0 → v2.2.0`처럼 둘째 숫자가 바뀌는 경우는 마이너 릴리즈입니다.
- `v2.2.0 → v2.2.1`처럼 셋째 숫자가 바뀌는 경우는 패치 릴리즈입니다.
- 핫픽스는 별도의 네 번째 숫자 단계가 아니라, 계산 오류처럼 빠르게 바로잡아야 하는 문제를 처리한 긴급 패치 릴리즈라는 뜻입니다.

기존 아카이브의 `메이저 패치`, `마이너 패치`, `버그 패치`라는 표기는 비개발자도 이해하기 쉬운 역사적 명칭으로 유지합니다. 숫자 체계는 위 기준과 일치합니다.

## 어떻게 확인했나요?

- 기본 전체 자동 검사에서 72개 검사 파일의 392개 항목이 통과했습니다. 외부 OpenAI 호출이나 별도 데이터베이스가 있어야 실행되는 7개 항목은 이 기본 검사에서 건너뛰었습니다.
- 피드백 전용 PostgreSQL은 로컬의 빈 일회용 데이터베이스에서 마이그레이션 3개를 처음부터 적용했습니다. AI 참조 문서 검색(RAG) 테이블이 섞이지 않는지, 180일 보관 기본값, 원문을 받지 않는 열 구성, 24시간 제한 테이블을 확인했습니다.
- 같은 일회용 데이터베이스에서 피드백 저장·수정·삭제·만료 정리와 동시 제한 검사 2개가 통과했습니다. 한 세션이 동시에 11번 요청했을 때 10번만 허용하고 1번은 거부하는 결과를 확인했습니다.
- 피드백 기능이 켜진 운영 빌드는 전용 데이터베이스 마이그레이션을 먼저 적용하고, 실패하면 새 버전을 배포하지 않도록 구성했습니다. 방화벽 규칙 게시와 `AI_FEEDBACK_EDGE_RATE_LIMIT_READY=1` 설정도 운영 프로젝트에서 확인했습니다.
- 다섯 언어의 참조 문서 1,015개와 OpenAI 검색 벡터 6,629개가 모두 최신 상태였고, 오래된 벡터는 0개였습니다. AI 답변 기준 검사 101개도 통과했습니다. 이번 패치는 AI 모델·지침·코퍼스를 바꾸지 않았으므로 새 유료 OpenAI 호출은 추가하지 않았습니다.
- 다섯 언어 제목·설명, 공식 대표 주소, 언어 대체 주소, 검색엔진 사이트맵, 구조화 데이터 검사가 통과했습니다. Vercel 기본 주소가 공식 대표 주소를 덮어쓰지 못하는 경우도 별도로 검사했습니다.
- TypeScript 검사와 코드 형식 검사가 통과했고, 배포용 화면 222개가 정상 생성됐습니다. 비밀정보 검사와 운영·개발 의존성 검사에서도 알려진 취약점은 0개였습니다.

## 공개 범위와 남아 있는 한계

- 계산식, 구조요약 출력, AI 모델, 참조 코퍼스와 벡터는 변경하지 않았습니다.
- 데스크톱·모바일 UI의 배치, 조작 방식, 스타일은 변경하지 않았습니다. 검색·공유용 메타데이터와 개인정보처리방침의 보안 설명만 갱신했습니다.
- 앱 안의 세션 제한은 정상적인 한 세션의 반복 요청을 막고, 운영 배포 앞단의 방화벽은 세션을 계속 새로 만드는 반복 요청을 IP 단위로 한 번 더 제한합니다.
- 피드백 기능이 켜진 운영 빌드는 전용 데이터베이스 마이그레이션, 게시된 Vercel Firewall 제한, `AI_FEEDBACK_EDGE_RATE_LIMIT_READY=1`을 모두 요구합니다. 이 조건 중 하나라도 빠지거나 마이그레이션이 실패하면 배포가 중단됩니다.
- 자동 검사는 프로그램 동작과 정보 노출 경계를 확인하며, 임상적 정확성이나 모든 미래 AI 답변을 보증하지 않습니다.
