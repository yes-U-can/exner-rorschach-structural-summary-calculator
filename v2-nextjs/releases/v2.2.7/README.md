## 주요 수정사항

### 개요

채점표에서 세 가지 미완성 입력이 그대로 계산으로 이어질 수 있던 문제를 수정했습니다.

- **[Location]**에서 흰 여백 반응을 단독 `S`로 선택할 수 있었습니다. 이제 위치 선택지는 `W`, `WS`, `D`, `DS`, `Dd`, `DdS`만 제공합니다.
- **[Determinants]**에서 같은 운동 계열의 부호를 한 반응에 두 개 이상 넣을 수 있었습니다(예: `Ma`와 `Mp`). 이제 한 계열에서 하나를 고르면 같은 계열의 다른 부호는 선택할 수 없습니다.
- **[FQ]**를 비워 둔 채 계산할 수 있었습니다. 이제 `+`, `o`, `u`, `-`, `none` 중 하나를 선택해야 계산할 수 있습니다.

모든 반응이 순수 형태(`F`)인 기록에서는 Lambda를 무한대 기호(`∞`) 대신 순수 F 반응의 개수로 보고합니다.

위치, 운동 결정인, 형태질을 규칙대로 입력해 온 기존 프로토콜은 이 문제의 영향을 받지 않으며 다시 계산할 필요가 없습니다. 과거 자동저장 자료에 단독 `S`, 같은 운동 계열의 중복 부호, 비어 있는 형태질이 남아 있으면 앱은 원래 입력을 보존한 채 계산을 멈추고, 확인이 필요한 행을 다섯 언어 안내로 알려줍니다.

### 왜 이 문제가 중요했나요?

**단독 `S`.** Exner 종합체계에서 흰 여백 사용은 독립된 위치가 아니라 기본 위치에 붙는 표기이며, 반응은 항상 `WS`, `DS`, `DdS`처럼 기록합니다. 과거 버전에서 위치를 `S` 하나로만 입력하면 흰 여백 빈도에는 집계되지만 `W`, `D`, `Dd` 어느 기본 위치에도 잡히지 않아, `W:D:Dd`와 `WDA%`처럼 기본 위치를 사용하는 값이 실제보다 낮게 계산될 수 있었습니다.

**같은 운동 계열의 중복.** 개별 반응의 운동 결정인은 계열마다 능동형 `a`, 수동형 `p`, 능동-수동형 `a-p` 중 하나로 기록합니다. 서로 다른 두 대상이 각각 능동·수동 움직임을 보이면 `Ma`와 `Mp`를 따로 넣는 것이 아니라 `Ma-p` 하나로 기록합니다. 과거 버전에서 `Ma`와 `Mp`를 별도 칸에 함께 넣으면 사람운동 빈도가 두 번 집계되어 EB의 왼쪽 값, EA, `a:p`, `Ma:Mp` 등이 실제보다 높게 계산될 수 있었습니다. 한 대상이 두 성질의 움직임을 함께 보였다는 이유만으로 자동으로 `a-p`가 되지는 않으며, 어떤 움직임이 부호화를 결정하는지는 반응기록과 질문단계(Inquiry)로 확인합니다.

**비어 있는 형태질.** `none`은 형태를 근거로 채점하지 않는 반응에 쓰는 정식 형태질 범주이고, 공란은 아직 채점을 마치지 않은 입력입니다. 과거 버전에서 형태질을 비워 두면 그 반응은 형태질 합계 어디에도 잡히지 않으면서 전체 반응 수에는 포함되어, `XA%`, `X+%`, `WDA%` 같은 값이 실제보다 낮게 계산될 수 있었습니다. 형태가 채점 근거가 되지 않는 반응이라면 이제 `none`을 직접 선택합니다.

**모든 반응이 순수 F인 Lambda.** Lambda는 `순수 F ÷ (전체 반응 − 순수 F)`이므로 모든 반응이 순수 F이면 분모가 0이 됩니다. 화면에는 무한대 기호 대신 순수 F의 개수를 보고합니다. 예를 들어 17개 반응이 모두 순수 F이면 `17.00`으로 표시합니다. 이 경계는 표준 실시에서 사실상 나타나지 않으며, 무한대 기호를 임상 보고 값으로 쓰지 않기 위해 채택한 소프트웨어 표기 방식입니다.

### 함께 수정한 내용

- 다섯 언어 참조 문서의 위치 `S`, 운동 결정인, 형태질 문서에 위 입력 규칙을 같은 내용으로 명시했습니다.
- 다섯 언어 문서가 같은 임상 규칙을 진술하는지 자동으로 검사하는 규칙 목록을 도입했습니다. 이제 한 언어의 문서만 고치고 다른 언어를 빠뜨리면 문서 생성 단계에서 검사가 실패합니다.
- 코딩 도우미가 단독 `S`, 같은 운동 계열의 중복, 비어 있는 형태질을 완성된 부호처럼 안내하지 않도록 답변 규칙을 보강했습니다.
- 참조 문서를 바꾼 뒤 다섯 언어 검색 자료와 OpenAI 임베딩을 모두 다시 만들었습니다.
- 스페인어·포르투갈어 문서의 오탈자와 용어 표기, 일본어 문서 네 곳에 남아 있던 영어 제목을 정리했습니다.
- 참조 문서 원고에서 실제로 사용되지 않던 상태 표시를 제거하고, 문서 상태 값의 허용 범위 검사를 추가했습니다.

## 테스트 및 검증

계산 규칙은 공개된 자료로 확인했습니다.

- 공식 RIAP v5 예제 보고서의 채점 계열에서 흰 여백이 항상 `WS`, `DdS`처럼 기본 위치와 결합해 표기되는 것을 확인했습니다.
- 같은 예제에서 형태질 표에 `none`이 정식 범주로 집계되는 것과, S-CON이 14세 초과 대상에게 적용된다는 명시를 확인했습니다.
- 운동 결정인의 능동·수동 규칙과 `a-p`의 적용 조건은 Exner 원전 내용을 옮긴 공개 임상 문헌으로 확인했습니다.
- 공개된 20반응 사례의 구조요약 값이 이번 패치 전후로 동일하게 계산되는 것을 확인했습니다.

입력 차단과 다섯 언어 안내도 실제 화면에서 확인했습니다.

- 위치 선택지에 단독 `S`가 나타나지 않습니다.
- 운동 계열에서 하나를 고르면 같은 계열의 다른 부호는 선택할 수 없게 표시됩니다.
- 형태질이 비어 있으면 계산이 멈추고 확인이 필요한 행을 알려줍니다. 다섯 언어의 안내 문구를 각각 실제 화면에서 확인했습니다.
- 과거 자동저장 자료에 남은 단독 `S`, 중복 운동 부호, 빈 형태질도 원본을 바꾸지 않고 같은 방식으로 안내합니다.

다섯 언어 검색 자료와 AI 도우미도 확인했습니다.

- 참조 문서 검색 질문 380개가 모두 관련 문서를 찾았습니다.
- OpenAI 임베딩 5,604개를 새 본문으로 다시 만들었고, 본문 해시 불일치와 오래된 임베딩은 0개였습니다.
- 실제 임베딩을 사용한 혼합 검색에서 넓은 질문과 이름을 지정한 질문의 상위 문서 적중률이 모두 100%였습니다.
- 단독 `S`, 운동 부호 중복, 빈 형태질을 묻는 대표 질문을 다섯 언어로 실제 호출해, 15회 모두 규칙에 어긋난 답변이 없는 것을 확인했습니다.
- 전체 자동 검사에서는 83개 테스트 파일의 471개 검사가 통과했고, 실행 조건이 없는 7개는 건너뛰었습니다. 배포용 빌드, 코드 정적 검사, 다섯 언어 문구 감사, 비밀정보 검사도 통과했습니다.

Lambda의 순수 F 개수 표기는 여러 공개 자료가 일치하는 소프트웨어 보고 방식이지만, 이를 명시한 원전 쪽수는 아직 확보하지 못했습니다. 확인한 범위와 남은 한계는 검증 문서에 그대로 적었습니다.

구현과 반복 검사에는 OpenAI Codex를, 릴리즈 전 문서·계산 근거 검토에는 Claude Fable 5를 사용했습니다. 도구 사이의 의견 일치를 정답으로 삼지 않았으며, 공개 전문 자료와 재현 가능한 계산 결과를 기준으로 삼았습니다.

## UI/UX, 개인정보와 계산 범위

- 새 화면이나 새 입력 항목을 추가하지 않았습니다.
- [Location] 선택지에서 단독 `S`를 제거했고, 같은 운동 계열의 중복 선택은 비활성 표시로 안내합니다.
- 규칙대로 입력된 기존 프로토콜의 계산 결과는 변하지 않습니다.
- 새로운 개인정보를 수집하지 않습니다.
- 채점 자료와 API 키를 서버 데이터베이스에 저장하지 않는 기존 원칙을 유지합니다.

## 공개 범위와 보안 경계

공개 소스에는 입력 경계 검사, 계산 회귀 검사, 다섯 언어 참조 문서와 규칙 동등성 검사 목록, 검색 자료 스냅샷을 포함합니다.

운영 환경변수, API 키, 실제 AI 문답 원문, 비공개 검토 기록, 로컬 경로는 공개하지 않습니다.
