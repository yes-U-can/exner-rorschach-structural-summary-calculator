# [2026-07-23] v2.2.7 バグ修正

[한국어](./README.md) | [English](./README.en.md) | [日本語](./README.ja.md) | [Español](./README.es.md) | [Português (Brasil)](./README.pt-BR.md)

## 主な修正内容

### 概要

採点表で三つの未完成の入力がそのまま計算に進む可能性があった問題を修正しました。

- **[Location]**では、白地を使う反応を単独の`S`として選択できました。現在、位置の選択肢は`W`、`WS`、`D`、`DS`、`Dd`、`DdS`のみを提供します。
- **[Determinants]**では、同じ運動系列の符号を一つの反応に二つ以上入力できました（例：`Ma`と`Mp`）。現在は、一つの系列で一つを選ぶと、同じ系列の他の符号は選択できません。同じ決定因そのものを二つの欄に繰り返し入力することも、あわせて遮断します（例：`FC`を二回）。
- **[FQ]**を空欄のままでも計算できました。現在は、`+`、`o`、`u`、`-`、`none`のいずれかを選択しなければ計算できません。

すべての反応が純粋形態（`F`）である記録では、Lambdaを無限大記号（`∞`）ではなく純粋F反応の個数として報告します。

位置、運動決定因、形態水準を規則どおり入力してきた既存のプロトコルは、この問題の影響を受けず、再計算の必要もありません。過去の自動保存データに単独の`S`、同じ運動系列の重複符号、空欄の形態水準が残っている場合、アプリは元の入力を保持したまま計算を止め、確認が必要な行を五言語の案内で知らせます。

### なぜ重要な問題だったのでしょうか？

**単独の`S`。** エクスナー包括システムでは、白地の使用は独立した位置ではなく基本の位置に付ける表記であり、反応は常に`WS`、`DS`、`DdS`のように記録します。過去のバージョンで位置を`S`のみとして入力すると、白地の頻度には集計されるものの、`W`、`D`、`Dd`のいずれの基本位置にも含まれず、`W:D:Dd`や`WDA%`のように基本位置を用いる値が実際より低く算出される可能性がありました。

**同じ運動系列の重複。** 個々の反応の運動決定因は、系列ごとに能動`a`、受動`p`、能動-受動`a-p`のいずれか一つで記録します。異なる二つの対象がそれぞれ能動・受動の動きを示す場合は、`Ma`と`Mp`を別々に入力するのではなく、`Ma-p`一つで記録します。過去のバージョンで`Ma`と`Mp`を別々の欄に一緒に入力すると、人間運動の頻度が二回集計され、EBの左側の値、EA、`a:p`、`Ma:Mp`などが実際より高く算出される可能性がありました。一つの対象が二つの性質の動きを同時に示したという理由だけで自動的に`a-p`になるわけではなく、どの動きが符号化を決定するかは、反応記録と質問段階（Inquiry）で確認します。同じ決定因そのものを二回入力した場合も該当する値が二回集計される可能性があったため、一つの反応では同じ決定因を一度だけ記録するよう、あわせて遮断しました。

**空欄の形態水準。** `none`は、形態を根拠として採点しない反応に用いる正式な形態水準の区分であり、空欄はまだ採点を終えていない入力です。過去のバージョンで形態水準を空欄のままにすると、その反応は形態水準のどの合計にも含まれないまま全反応数には含まれ、`XA%`、`X+%`、`WDA%`のような値が実際より低く算出される可能性がありました。形態が採点の根拠にならない反応であれば、現在は`none`を直接選択します。

**すべての反応が純粋FであるLambda。** Lambdaは`純粋F ÷（全反応 − 純粋F）`であるため、すべての反応が純粋Fの場合は分母が0になります。画面には無限大記号ではなく純粋Fの個数を報告します。たとえば17個の反応がすべて純粋Fであれば、`17.00`と表示します。この境界は標準的な実施では事実上現れず、無限大記号を臨床報告の値として使わないために採用したソフトウェア上の表記方式です。

### 関連する修正

- 五言語の参照文書のうち、位置`S`、運動決定因、形態水準の文書に、上記の入力規則を同じ内容で明記しました。
- 五言語の文書が同じ臨床規則を記述しているかを自動的に検査する規則リストを導入しました。今後は、一つの言語の文書だけを修正して他の言語を修正し忘れると、文書生成の段階で検査が失敗します。
- コーディング支援AIが、単独の`S`、同じ運動系列の重複、空欄の形態水準を完成した符号のように案内しないよう、回答規則を強化しました。
- 参照文書の変更後、五言語の検索データとOpenAI埋め込みをすべて再作成しました。
- スペイン語・ポルトガル語文書の誤字と用語表記、日本語文書の四か所に残っていた英語の見出しを整理しました。
- 参照文書の原稿で実際には使用されていなかった状態表示を削除し、文書の状態値の許容範囲検査を追加しました。

## テストと検証

計算規則は公開資料で確認しました。

- 公式RIAP v5のサンプル報告書の採点系列で、白地が常に`WS`、`DdS`のように基本位置と結合して表記されていることを確認しました。
- 同じサンプルで、形態水準の表に`none`が正式な区分として集計されていることと、S-CONが14歳を超える対象に適用されると明記されていることを確認しました。
- 運動決定因の能動・受動の規則と`a-p`の適用条件は、エクスナー原典の内容を伝える公開臨床文献で確認しました。
- 公開されている20反応の事例の構造一覧表の値が、このパッチの前後で同一に計算されることを確認しました。

入力の遮断と五言語の案内も実際の画面で確認しました。

- 位置の選択肢に単独の`S`は表示されません。
- 運動系列で一つを選ぶと、同じ系列の他の符号は選択できない状態で表示されます。
- どの決定因でも、すでに選択した符号は他の欄では選択できない状態で表示されます。
- 形態水準が空欄の場合は計算が止まり、確認が必要な行を知らせます。五言語の案内文をそれぞれ実際の画面で確認しました。
- 過去の自動保存データに残る単独の`S`、重複した運動符号、空欄の形態水準も、元のデータを変更せずに同じ方式で案内します。

五言語の検索データとAIアシスタントも確認しました。

- 参照文書を検索する380個の質問が、すべて関連文書を取得しました。
- OpenAI埋め込み5,604個を新しい本文から再作成し、本文ハッシュの不一致と古い埋め込みは0個でした。
- 実際の埋め込みを用いたハイブリッド検索では、広い質問と名称を指定した質問の上位文書的中率がいずれも100%でした。
- 単独の`S`、運動符号の重複、空欄の形態水準を尋ねる代表質問を五言語で実際に呼び出し、15回すべてで規則に反する回答がないことを確認しました。
- 自動検査全体では、83個のテストファイルにある476個の検査が合格し、実行条件のない7個はスキップしました。デプロイ用ビルド、コードの静的検査、五言語の文言監査、秘密情報検査も合格しました。

Lambdaの純粋F個数表記は、複数の公開資料が一致するソフトウェア上の報告方式ですが、これを明記した原典のページ番号はまだ確保できていません。確認した範囲と残る限界は、検証文書にそのまま記録しました。

実装と反復検査にはOpenAI Codexを、リリース前の文書・計算根拠の検討にはClaude Fable 5を使用しました。ツール間の意見一致を正解とはせず、公開専門資料と再現可能な計算結果を基準としました。

## UI/UX、プライバシー、計算範囲

- 新しい画面や入力項目は追加していません。
- [Location]の選択肢から単独の`S`を削除し、同じ運動系列の重複選択は無効表示で案内します。
- 規則どおり入力された既存プロトコルの計算結果は変わりません。
- 新たな個人情報は収集しません。
- 採点データとAPIキーをサーバーデータベースに保存しない従来の原則を維持します。

## 公開範囲とセキュリティ境界

公開ソースには、入力境界の検査、計算回帰検査、五言語の参照文書と規則同等性の検査リスト、検索データのスナップショットを含めます。

運用環境変数、APIキー、実際のAI問答の原文、非公開の検討記録、ローカルパスは公開しません。
