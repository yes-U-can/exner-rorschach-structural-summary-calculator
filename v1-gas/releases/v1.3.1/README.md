### *버그 수정*

> **샘플 데이터 로드 시 슬롯 가시성 복원 문제**
>
- **문제 상황**
    - 샘플 데이터를 불러올 때 Determinants, Contents, Special Score에서

        여러 개의 항목이 입력되어 있는데 하나의 드롭다운만 표시되는 문제 발생

    - 예: `det4_1`, `det4_2`, `det4_3`에 값이 있어도 첫 번째 슬롯만 표시됨
- **원인 분석**
    - `sampleData` 객체에 슬롯 가시성 메타데이터(`_detX_visible`, `_conX_visible`, `_ssX_visible`)가 없음
    - `loadData()` 함수가 메타데이터가 없을 경우 기본값 1만 사용하여 첫 번째 슬롯만 표시
- **해결 방법**
    - `loadData()` 함수에서 메타데이터가 없거나 유효하지 않을 경우, 실제 데이터를 확인하여 값이 있는 가장 높은 슬롯 인덱스를 자동으로 찾도록 수정
    - 예: `det4_1`, `det4_2`, `det4_3`에 값이 있으면 자동으로 `maxVisibleIndex = 3`으로 설정하여 3개 슬롯 모두 표시
- **기술적 세부사항**
    - `loadData()` 함수 내 슬롯 가시성 복원 로직 개선 (2733-2760번 줄)
    - 메타데이터가 없을 경우 실제 데이터를 순회하여 최대 슬롯 인덱스 계산
    - Determinants, Contents, Special Score 모두에 동일한 로직 적용
- **파일**: `index.html` (2733-2760번 줄)

> **입력값 초기화 시 슬롯 가시성 및 버튼 복원 문제**
>
- **문제 상황**
    - 입력값 초기화 버튼을 누르면 모든 행의 입력값은 초기화되지만,

        슬롯 가시성이 초기화되지 않아 여러 개의 드롭다운 메뉴가 남아있는 문제 발생

    - 특히 샘플 데이터를 불러온 후 초기화하면,

        여러 개의 슬롯이 있던 행에서 +/- 버튼이 사라지는 문제 발생

    - 예: 5번 행의 Determinants에 6개 슬롯이 있었던 경우,

        초기화 후에도 6개 슬롯이 남아있고 버튼이 사라짐

- **원인 분석**
    - `performReset()` 함수에서 입력값만 초기화하고 슬롯 가시성을 초기화하지 않음
    - 슬롯 가시성을 초기화한 후에도 버튼 위치를 복원하지 않아 버튼이 마지막 슬롯에 남아있거나 사라짐
- **해결 방법**
    - `performReset()` 함수에 모든 행(1~50)의 슬롯 가시성 초기화 로직 추가
        - 각 그룹(det, con, ss)의 모든 슬롯(1~6)을 숨기고 첫 번째 슬롯만 표시
    - 1~15번 행을 표시할 때 각 그룹에 대해 `updateSlotButtons()` 함수를 호출하여 버튼을 첫 번째 슬롯으로 복원
- **기술적 세부사항**
    - `performReset()` 함수 내 슬롯 가시성 초기화 로직 추가 (2353-2376번 줄)
    - 버튼 복원 로직 추가 (2384-2386번 줄)
    - 모든 행(1~50)에 대해 슬롯 초기화를 수행하여 어떤 상태에서든 완전한 초기화 보장
- **파일**: `index.html` (2340-2401번 줄)
