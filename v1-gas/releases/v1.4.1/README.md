# 주요 수정사항

## *개요*

> **v1.4.0의 버그를 수정했습니다.**

웹앱에 처음 접속했을 때와 재접속했을 때 표시되는 버튼이 올바르게 나타나도록 수정했습니다.
>
- 이전에는 재접속해도 "샘플 데이터로 시작하기" 버튼이 표시되는 버그가 있었습니다.
    - 처음 접속: "샘플 데이터로 시작하기" 버튼 표시
    - 재접속 (저장된 데이터 있음): "자동저장된 내용 불러오기" 버튼 표시
- 재접속 시 모달 화면을 더 깔끔하게 보이도록 개선했습니다.
    - "작업 이어하기" 제목을 제거하고 "마지막에 작업했던 내용이 있습니다." 메시지만

        굵은 큰 글씨로 표시 (불필요하게 두 번 말하지 않도록 개선)


> 기타 개선사항
>
- 더보기 탭의 카드에 마우스를 올렸을 때 더 눈에 띄는 효과를 추가했습니다.
    - 그림자가 더 진하게 나타남
    - 카드 뒤의 배경 이미지가 더 선명하게 보임
    - 카드 안의 글씨는 흐리게 처리되어 배경 이미지가 더 돋보임
- `styles.html`에 있어야 할 스타일이 일부 `index.html`에 남아있던 것을 모두 옮겨서

    코드를 더 깔끔하게 정리했습니다.


또한 일련의 변경점은 [Gems 챗봇](https://gemini.google.com/gem/1QDCPHshPvq5J9iIKeV-1Nvy0EFzKPN6Y?usp=sharing)에 반영되었습니다.

## *세부사항*

### *초기 접속 모달 버튼 표시 로직 수정*

> **구현 내용**
>
- **문제 상황**
    - 최초 접속 시와 재접속 시 모두 "샘플 데이터로 시작하기" 버튼이 표시되는 문제 발생
    - 재접속 시 "샘플 데이터로 시작하기" 버튼을 클릭해도 실제로는 자동저장된 내용이 불러와지는 버그 발생
- **해결 방법**
    - 저장된 데이터가 있을 때: "자동저장된 내용 불러오기" 버튼 표시
    - 저장된 데이터가 없을 때: "샘플 데이터로 시작하기" 버튼 표시
    - `updateAllTexts()` 함수가 모달이 이미 표시된 상태에서는 텍스트를 덮어쓰지 않도록 수정
    - 모든 지원 언어(한국어, 영어, 일본어, 스페인어, 포르투갈어)에서 동일하게 동작하도록 개선

> **기술적 세부사항**
>
- `updateAllTexts()` 함수에서 모달이 이미 표시되어 있으면 모달 텍스트를 업데이트하지 않도록 가드 추가
- `handleScoringTabFirstLoad()` 함수에서 저장된 데이터 유무를 확인하여 모달 텍스트 설정
- 저장된 데이터가 있을 때 `modalLoadBtn.textContent`를 `t('modal_welcome_load_saved')`로 설정
- 저장된 데이터가 없을 때 `modalLoadBtn.textContent`를 `t('modal_welcome_load')`로 설정
- `updateAllTexts()` 함수 내부에서도 저장된 데이터 유무를 확인하여 모달 텍스트 설정 (모달이 표시되기 전에 실행되는 경우 대비)

**파일**: `index.html` (2387-2426번 줄: updateAllTexts 함수, 3974-4060번 줄: handleScoringTabFirstLoad 함수)
