## *세부사항*

### *행(Row) 제어 시스템*

> **구현 내용**
>
- **초기 상태**: 15개 행만 표시, 나머지 35개는 숨김 처리
- **최소/최대 제한**: 최소 1개, 최대 50개 행
- **제어 방식**: CSS `visibility` 토글 방식 (DOM 조작 없음)
- **데이터 무결성**: 숨겨진 행의 모든 필드 자동 초기화

> **유효성 검증**
>
- **R < 14 경고**: 반응 수가 너무 적을 때 토스트 메시지 표시
- **R ≥ 45 경고**: 반응 수가 과다할 때 경고 메시지 표시 (수식 표기: `R ≥ 45`)

> **사용자 안내**
>
- 행 추가/삭제 버튼 옆 정보 아이콘 추가
- 툴팁 메시지: "최하단의 행에 대해서만 조작 가능하며, 최대 50개까지 가능합니다."
- 추가 팁: "반응 행의 순서는 결과 계산에 영향을 미치지 않습니다"

### *열(Column) 제어 시스템*

> **동적 슬롯 시스템**
>
- **대상 열**: Determinants, Contents, Special Scores
- **구현 방식**
    - 각 변인별로 초기 1개 슬롯만 표시
    - "+" 버튼으로 슬롯 추가 (최대: Determinants/Contents 6개, Special Scores 8개)
    - "-" 버튼으로 마지막 슬롯 제거
    - 모든 슬롯은 미리 렌더링되어 있으며, CSS 클래스로 표시/숨김 제어
- **슬롯 배치**
    - 각 슬롯은 드롭다운 메뉴 바로 아래에 수직으로 배치
    - +/- 버튼은 마지막 표시된 슬롯 하단에 고정 배치
    - 테이블 헤더 너비 변경 없이 안정적인 레이아웃 유지

> **경고 시스템**
>
- **Determinants/Contents ≥ 4**: "과잉 채점 주의" 토스트 메시지
- **Special Scores ≥ 5**: "과잉 채점 주의" 토스트 메시지
- 경고 메시지에 임상적 가이드라인 포함 (예: 형태($F$) 중복 채점 확인)

> **데이터 처리**
>
- 빈 슬롯은 계산에서 자동 제외
- 숨겨진 슬롯의 값은 자동으로 `null` 또는 `""`로 초기화

### *입력 필드 개선*

> **Response 필드**
>
- **변경 사항**: `input type="text"` → `textarea rows="2"`
- **기능**: 다중 줄 입력 지원, 자동 줄바꿈
- **계산 영향**: 계산에 포함되지 않음 (메모 용도)

> **Score 및 G/PHR 필드**
>
- **자동 계산**: Card와 Z 값 선택 시 자동 계산
- **사용자 입력 불필요**: 직접 입력 불가 (읽기 전용)
- **안내 메시지**: 채점 영역 상단에 명확한 안내 문구 추가

### *유효성 검증 시스템 강화*

> **클라이언트 사이드 검증**
>
- **필수 입력 항목** (활성화된 행 중 하나라도 비어있으면 오류)
    - Card
    - Location
    - DQ
    - Determinants (최소 1개 슬롯에 값 필요)
    - FQ
    - Contents (최소 1개 슬롯에 값 필요)
- **제외 항목** (비어있어도 계산 가능)
    - Response
    - Score
    - G/PHR
    - Pair
    - P
    - Z
    - Special Score

> **오류 메시지**
>

```
생성된 반응 중 채점 내용이 입력되지 않은 행이 있습니다.

다시 한 번 확인해주세요.
반드시 채워넣어야 하는 항목:

Card / Location / DQ / Determinants / FQ / Contents

```

### *데이터 로딩 개선*

> **다중 슬롯 데이터 로딩**
>
- **문제**:

    샘플 데이터 및 저장된 데이터에서

    Determinants, Contents, Special Scores의 다중 값이 1개만 로딩되는 문제

- **해결**:

    `loadData` 함수 수정하여

    모든 가능한 슬롯(`det${i}_${j}`, `con${i}_${j}`, `ss${i}_${j}`)을 순회하며 값 설정 및 가시성 조정


> **행 추가 후 슬롯 초기화**
>
- **문제**: 계산 후 행 추가 시 Determinants, Contents, Special Scores가 비어있는 문제
- **해결**: `addRow` 함수에서 슬롯 가시성 재설정 및 값 초기화, 이벤트 리스너 재등록

> **드롭다운 중복 옵션 제거**
>
- **문제**: Determinants, Contents, Special Scores 드롭다운에 빈 옵션이 2개 표시
- **해결**: `createVariableSlot` 함수에서 `optionsList.filter(o => o !== '')` 적용

GAS 프로젝트에서 웹앱으로 배포하면 해당 버전의 계산기를 직접 실행할 수 있습니다.
